<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User text input → profile data parser | Free text parsed into structured data (age, canton, salary) |
| Consent chip tap → ConsentManager | User intent translated to consent grant/revoke |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | ChatDataCaptureHandler.parseAge | mitigate | Validate parsed age is 0-150 range, reject non-numeric. parseSalary rejects negative values. parseCanton validates against known Swiss canton list. |
| T-03-05 | Information Disclosure | consent grant without informed choice | mitigate | Each consent sentence is human-readable, specific about what is shared. Accept/decline chips are equally prominent (no dark pattern). Decline is always respected immediately. |
| T-03-06 | Spoofing | user types "oui" in data-capture mode interpreted as consent | mitigate | Data capture mode and consent mode are distinct states. Free text during consent mode is ignored (only chip taps count). |
| T-03-07 | Elevation of Privilege | tone preference bypasses voice cursor contract | accept | VoicePreference enum has only 3 values. setVoiceCursorPreference validates the enum. No way to inject arbitrary cursor values. |
</threat_model>
