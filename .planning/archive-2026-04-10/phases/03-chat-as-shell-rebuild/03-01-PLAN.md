<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM → route_to_screen tool call | Untrusted route string from LLM could attempt to open arbitrary screens |
| User tap → suggestion chip → route | User-initiated but route string comes from i18n mapping |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | route_to_screen tool call | mitigate | resolveDrawerWidget validates route against MintScreenRegistry + ToolCallParser whitelist (existing T-02-01 mitigation preserved). Unknown routes return null, silently dropped. |
| T-03-02 | Denial of Service | showChatDrawer spam | mitigate | ChatToolDispatcher._maxToolCallsPerResponse (existing cap at 5) prevents LLM from flooding drawers. Only one bottom sheet can be open at a time (Flutter constraint). |
| T-03-03 | Information Disclosure | drawer shows authenticated data to anonymous user | accept | Drawers show calculator screens which are read-only projections based on user-entered data. No server-side auth data exposed. Scope guard (Phase 1) prevents route-level leaks. |
</threat_model>
