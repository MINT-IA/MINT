# Phase 1.5 Plan: chiffre_choc → premier_eclairage Domain Rename Summary

Full three-layer domain rename of the legacy `chiffre_choc` token (Python identifiers + JSON wire format + URL routes) across the entire live MINT codebase, including backend, mobile, ARB, OpenAPI, RAG corpus, and live docs, with a new CI grep gate enforcing zero residue.

## Deviations from Plan

### Scope amendments mid-execution

**1. Gate protocol rebaseline (Option 3, user-approved at Step 0.5)**

The original D-07 gate protocol (`flutter test` aggregate ≥ 8137, strict zero-failure) was unimplementable because the **baseline on HEAD of `feature/v2.2-p0a-code-unblockers` before any Phase 1.5 change already had 3 failing flutter tests and 6 pre-existing ruff errors** (see `01.5-BASELINE-RED.md`). A strict protocol would have infinite-looped on reset.

User approved Option 3:
- Flutter test: allowlist semantics. Aggregate ≥ 8991 passed, failures ⊆ {`test/patrol/onboarding_patrol_test.dart`, `test/patrol/document_patrol_test.dart`, `test/golden_screenshots/quick_start_screen_golden_test.dart`}.
- Ruff: skipped for Phase 1.5 (6 pre-existing E402 in `wizard.py` unrelated).
- Backend commits skip flutter test; end-of-phase full run covers it.
- F-layer commits run targeted subsets.
- 8991 becomes the new test-count floor (superseding 8137).

**2. 13-commit plan collapsed to 4 commits (compile-boundary atomicity)**

The original D-01 plan called for 13 atomic commits: B1-B4 (backend selector → services → endpoints → OpenAPI), F1-F5 (Flutter model → onboarding → life-event screens → services → widgets), L1 (ARB), A1 (analytics), G1 (CI gate), D1 (docs flip).

**Discovery during B1 execution:** neither Python nor Dart permits independent layering at the granularity the plan assumed.
- Python: `app/services/onboarding/__init__.py` eagerly re-exports the selector, which is transitively imported by `app/main.py` at app startup. Attempting B1 alone (selector + tests + init) left the endpoint layer broken on the removed symbol names, and pytest could not even import the conftest. The rename must be atomic across model + selector + services + schemas + endpoints + tests.
- Dart: the whole-library compile model means `flutter analyze lib/` must be 0 errors after every commit, and the mobile model layer (F1) is consumed by services (F4), widgets (F5), and screens (F2, F3) — all of which reference renamed symbols simultaneously. Splitting F1 alone would leave hundreds of analyze errors.

**Resolution:** Merged B1+B2+B3+B4 into one BE commit (114 files) and F1+F2+F3+F4+F5+L1+A1 into one MOB commit (145 files). G1 (CI gate creation) and D1 (docs sweep + CLAUDE.md flip) were then merged into a single G1+D1 commit because the gate must be wired AFTER the sweep (otherwise CI would break).

**3. Three scope adjustments from the PRE-AUDIT folded into MOB**

- F2 absorbed the onboarding-screen routing literal rename (`screenName: 'chiffre_choc'` → `'premier_eclairage'`).
- F4 updated the `api_service.dart:969` defensive fallback from `['premierEclairage', 'premier_eclairage']` to `['premierEclairage']` (single canonical wire key post-BE).
- F4 renamed the two coach routing literals in `coach_orchestrator.dart:882` and `prompt_registry.dart:502`, in lockstep with the BE commit's `ComponentType.premier_eclairage` enum value.

**4. URL route rename (Option A, user-approved mid-run)**

The original CONTEXT.md plan did not explicitly mention URL route changes. During B1 entry, the executor discovered that `/chiffre-choc` is a live URL route in `services/backend/app/api/v1/endpoints/onboarding.py` and `coach.py`, plus `/onboarding/chiffre-choc` and `/chiffre-choc-instant` are mobile go_router routes. User approved **Option A** (full three-layer rename, including URL routes), folded into BE and MOB. All references across live code are now `/premier-eclairage*`.

**5. Class name collision fix (PremierEclairageCard)**

Two widgets ended up with the same class name after the rename:
- `lib/widgets/onboarding/premier_eclairage_card.dart` → `class PremierEclairageCard extends StatefulWidget` (pre-existing partial rename)
- `lib/widgets/coach/chiffre_choc_card.dart` → renamed file to `premier_eclairage_card_coach.dart` to avoid filename collision, but class was also `PremierEclairageCard extends StatelessWidget`

Resolution: renamed the coach variant's class to `PremierEclairageCoachCard` and updated its single consumer (`premier_eclairage_section.dart`). Documented in the MOB commit message.

**6. Education/RAG corpus sweep (end-of-phase test failure)**

The first end-of-phase full flutter test run showed 3 new failures in `test/rag_knowledge_base_test.dart` and `test/services/rag_retrieval_service_test.dart`. Root cause: the docs-sweep in the pre-audit searched only for `chiffre_choc` (snake) but the `education/inserts/concepts/` and `education/inserts/faq/` markdown files used **"Chiffre Choc"** (with space, as a section header in user-facing French). The tests were forward-looking and expected `## Premier Éclairage`.

Fix: extended the sweep to cover `education/` and `legal/` markdown files (138 files updated). Added `Chiffre choc` variant to the replacement set. Re-ran the failing tests → green. Re-ran the full suite → back to exactly the baseline 3 allowlisted failures.

**7. `tools/openapi/openapi.json` regen path**

`tools/openapi/generate_canonical.py` only writes to `mint.openapi.canonical.json`. The separate `openapi.json` is a committed snapshot not auto-regenerated by the script. Regenerated directly via `python3 -c "from app.main import app; json.dump(app.openapi(), ...)"` ad-hoc call during BE. Documented for the next executor — this is a **latent issue in the repo** (two OpenAPI artifacts, only one regenerated by the canonical script) that should probably be resolved by consolidating the script.
