Per-note triage procedure:
1. Parse the row: `# | Run | Timestamp | Surface | Axis | Severity (Julien's call) | Description | Proposed fix`
2. **Re-apply D-04 literally** — do NOT rubber-stamp Julien's severity. Claude is the second opinion, specifically to catch:
   - Polish/nit calls that are actually blockers (shame leaks, banned terms, perf violations)
   - Block calls that are actually polish (Julien over-sensitive)
3. **Shame test:** if description mentions confusion, jargon, feeling behind, feeling stupid, feeling insulted → upgrade to block regardless of Julien's call.
4. **Banned terms check:** grep description for "garanti", "certain", "assuré", "sans risque", "optimal", "meilleur", "parfait", "conseiller" → if present and user-visible → block.
5. **Perf check:** any perf metric failing (cold_start ≥2500, interactive ≥3000, first_reply ≥4000) → automatic block.
6. **Dispatch target:** for each block, identify which Phase 7 (landing) or Phase 10 (onboarding) file the fix touches. If the blocker is a chat / first-insight issue, it may touch `claude_coach_service.py` or ARB files — legitimate, still hot-fixable under D-05.
