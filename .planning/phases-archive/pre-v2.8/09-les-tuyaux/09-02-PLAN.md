<objective>
Wrap the coach chat agent loop with a 55-second total deadline and 25-second per-iteration cap,
so users get a graceful timeout message instead of a 502 Bad Gateway when the LLM is slow.

Purpose: Railway's gateway timeout and Gunicorn's 120s worker timeout are both higher than the
UX ceiling (~60s). Without an application-level deadline, multi-turn tool calls can run 90+ seconds,
causing the user to see a blank error.
Output: Modified coach_chat.py with timeout wrapping + updated tests.
</objective>

        return {"answer": "slow answer", "tool_calls": [], "sources": [], "disclaimers": [], "tokens_used": 100}
    mock_orch.query = slow_iteration_query

    from app.api.v1.endpoints.coach_chat import _run_agent_loop
