# 27-01-PLAN — Stabilisation Critique

**Phase:** 27
**Plan:** 01
**Branch:** `feature/27-01-stabilisation-critique`
**Dependencies:** None (first plan of v2.7)

## Goal
Rendre le coach incassable en prod : MSG2 fiable 100%, retry Anthropic transparent, token budget adaptatif avec soft degradation (pas d'erreur visible), feature flags pour rollback instantané.

## Strategy (post deep research)

**Principe** : ne pas ré-écrire l'agent loop (risque trop élevé sur 1975 lignes). Wrapper observabilité + guards additionnels + graceful degradation chain.

**3 fixes clés** :
1. **Content-block inspection** — ne plus trust `stop_reason`, parser les blocks explicitement.
2. **Reflective retry** — si `answer_text=""` ET `stop_reason=end_turn` (bug Sonnet 4.5), force one more iteration sans tools.
3. **Graceful model fallback** — Sonnet fail (429/529/timeout) → Haiku auto, flag `degraded=true` dans logs.

**Tokens & flags** :
- Token budget journalier per user → soft cap (Sonnet→Haiku à 80%, compacter context à 95%, message calme à 100%).
- Feature flags Redis polling 60s (déjà Redis dans stack pour rate_limit).
- SLO-based auto-rollback via cron 30s.

### Task 2: Anthropic retry with tenacity
**File**: `services/backend/app/services/coach/claude_coach_service.py`

**Changes**:
- Add `tenacity>=8.2` to `services/backend/pyproject.toml` / `requirements.txt` (check current state first).
- Wrap the Claude API call (`anthropic.messages.create` or equivalent) with `@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=0.5, max=8), retry=retry_if_exception_type(...))` targeting `anthropic.APIStatusError` with status in {429, 500, 502, 503, 504, 529}.
- Log each retry attempt with `logger.warning("anthropic retry %d/3: status=%s", attempt_number, status)`.
- On final failure, raise custom `CoachUpstreamError` (new class in same file) so downstream can decide Haiku fallback vs user-facing message.

### Task 3: Model fallback chain (graceful degradation)
**File**: `services/backend/app/services/coach/claude_coach_service.py`

**Changes**:
- New helper `_call_with_fallback(messages, tools, system, primary_model, fallback_model)` that tries `primary_model` (Sonnet 4.6) → on `CoachUpstreamError` or timeout 20s → retries with `fallback_model` (Haiku 4.5) with truncated messages (keep system + last 10 turns max).
- Returns `{..., "degraded": bool, "model_used": str}` so orchestrator can log.
- Flag the Flutter response with `response_meta.degraded=true` (add field to `CoachChatResponse` schema).

### Task 4: Token budget per user/day + soft cap
**New file**: `services/backend/app/services/coach/token_budget.py`

**Contract**:
```python
class TokenBudget:
    DAILY_LIMIT_TOKENS = 50_000  # default
    SOFT_CAP_RATIO = 0.80        # → switch to Haiku
    TRUNCATE_RATIO = 0.95        # → truncate context
    HARD_CAP_RATIO = 1.00        # → calm decline

    async def consume(self, user_id: str, tokens: int) -> BudgetState: ...
    async def current_state(self, user_id: str) -> BudgetState: ...

@dataclass
class BudgetState:
    used: int
    limit: int
    ratio: float
    tier: Literal["normal", "soft_cap", "truncate", "hard_cap"]
    recommended_model: str
```

**Backing store**: Redis key `coach:budget:{user_id}:{YYYY-MM-DD}` with TTL 48h, atomic INCRBY.

**Integration**: coach_chat.py calls `budget.current_state()` BEFORE agent loop. Passes `recommended_model` to claude_coach_service. Calls `budget.consume(tokens)` after each iteration.

**Hard cap message** (i18n key `coach.budget.daily_limit_reached`): "On a déjà bien avancé aujourd'hui. Repose-toi, je t'attends demain." (aligned with doctrine "MINT doit se rendre inutile").

### Task 6: SLO-based auto-rollback
**New file**: `services/backend/app/services/slo_monitor.py`

**Contract**:
- Every 30s (background task via `fastapi_utils.tasks.repeat_every` or asyncio.create_task in `main.py` startup), compute:
  - `fallback_rate_5m` = count of coach_chat responses with `degraded=true` OR `answer=fallback_template` in last 5min / total.
  - `p95_latency_5m` = p95 of `total_duration_ms` in last 5min.
- If `fallback_rate_5m > 0.05` OR `p95_latency_5m > 5000ms` for 2 consecutive checks → auto-disable `COACH_FSM_ENABLED` via FlagsService + Sentry alert.

**Metrics source**: Sentry breadcrumbs OR new Redis metrics (simpler): `coach:metrics:{YYYY-MM-DD-HH-MM}` as Redis hash with counters.

## Risks & Mitigations
- **Risk**: tenacity retry on non-idempotent POST duplicates work. **Mitigation**: only retry on 429/5xx (server-side errors, original request not processed).
- **Risk**: Redis unavailable → flags/budget break. **Mitigation**: fail-open on flags (treat as disabled), fail-open on budget (allow request, log warning).
- **Risk**: model fallback to Haiku gives worse response → user notices. **Mitigation**: degraded chip is subtle, Haiku 4.5 is actually quite strong in FR for simple follow-ups.
- **Risk**: SLO auto-rollback creates flapping. **Mitigation**: require 2 consecutive breaches before rollback.
