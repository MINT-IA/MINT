<objective>
Fix the save_insight / retrieve_memories round-trip so the coach remembers facts across sessions.

Purpose: Gate 0 found coach doesn't remember salary/canton/decisions between sessions. Investigation reveals save_insight is ack-only (STAB-12 comment: "Persistence to the memory layer is a v3.0 item") — the LLM thinks it saved something but nothing is written to DB. Meanwhile retrieve_memories only searches the Flutter-sent memory_block text, not any persistent store. The fix: (1) create a CoachInsightRecord DB model, (2) persist save_insight calls to DB, (3) load saved insights into a new memory block section injected into the system prompt, (4) make retrieve_memories also search DB-persisted insights.

Output: Working cross-session memory where save_insight persists and insights load into the next conversation's system prompt.
</objective>
