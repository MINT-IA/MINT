# Milestone v2.5 Audit — Transformation

## Summary

**Status: GAPS FOUND**

6/6 phases complete with automated verification passing. But cross-phase integration check reveals **10 broken or unwired requirements** out of 28 total. The pattern is "facade sans cablage" — each phase is internally clean but 5 critical wiring joints are missing.

## P0 Findings (Broken Wires)

### P0-1: Partner estimate data never persists (COUP-01, COUP-02, COUP-03, COUP-04)
`save_partner_estimate` is in `INTERNAL_TOOL_NAMES` → backend handles internally → Flutter `widget_renderer.dart` intercept is dead code → SecureStorage never written → entire couple mode disconnected.

### P0-2: Pre-mortem handler does not persist to DB (CMIT-05, CMIT-06)
`save_pre_mortem` handler at `coach_chat.py:800-804` logs and acks but never calls `db.add()` / `db.commit()`. Pre-mortem data is permanently lost.

### P0-3: AnonymousIntentScreen is an orphan (ANON-02)
Intent screen with 6 pills exists but has NO GoRouter route. Users never see it. Entry point to anonymous flow is disconnected.

## P1 Findings (Missing Connections)

### P1-1: Conversation key mismatch (TIME-02, TIME-04)
TensionCardProvider/TimelineProvider read `'_chat_conversation_index'` but ConversationStore uses `'${userId}__chat_conversation_index'` for authenticated users. Conversation count always 0.

### P1-2: CoupleQuestionGenerator and degradeForPartnerEstimate are orphans (COUP-02, COUP-03)
Both defined and tested but never imported or called by any screen or service.

### P1-3: Fresh-start notifications never scheduled (CMIT-04)
`scheduleAllFreshStartNotifications()` exists but is never called from any init sequence.

## P2 Findings (Schema/Data Issues)

### P2-1: Commitment status mismatch
TensionCardProvider filters `status == 'active'` but backend only has `pending`/`completed`/`dismissed`.

### P2-2: Timeline month names hardcoded in French
i18n violation in `TimelineProvider._monthLabel()`.

## P3 Findings (Performance)

### P3-1: TimelineProvider double-fetches
`refresh()` calls `super.refresh()` then `_fetchRawData()` — both hit the same services.
