## Motion Contract

Source: `lib/theme/mint_motion.dart` — NEVER declare inline Duration/Curve values. Always use MintMotion tokens.

### Token Reference

| Token | Value | Usage |
|-------|-------|-------|
| `MintMotion.fast` | 150ms | Color changes, opacity toggles |
| `MintMotion.standard` | 300ms | Card expand, slide, fade — standard UI |
| `MintMotion.slow` | 600ms | Hero number reveal, graph draw |
| `MintMotion.page` | 350ms | Page transitions (CustomTransitionPage) |
| `MintMotion.curveStandard` | `Curves.easeOutCubic` | Most UI transitions |
| `MintMotion.curveEnter` | `Curves.easeOutQuart` | Entering elements |
| `MintMotion.curveExit` | `Curves.easeInCubic` | Exiting elements |

### UXP-01: Signal Card AnimatedSwitcher (Aujourd'hui Tab)

**Widget:** `AnimatedSwitcher` wrapping the signal card slot in `MintHomeScreen`

**Spec:**
```dart
AnimatedSwitcher(
  duration: MintMotion.standard,          // 300ms
  reverseDuration: MintMotion.fast,       // 150ms exit
  switchInCurve: MintMotion.curveEnter,   // Curves.easeOutQuart
  switchOutCurve: MintMotion.curveExit,   // Curves.easeInCubic
  transitionBuilder: (child, animation) => FadeTransition(
    opacity: animation,
    child: child,
  ),
  child: signalCard,  // must have unique Key per card type
)
```

**Key requirement:** Each distinct card type must carry a `ValueKey<String>` (e.g., `ValueKey('plan_reality')`, `ValueKey('premier_eclairage')`) — otherwise AnimatedSwitcher will not detect a child swap.

**Behaviour contract:**
- Card dismissed → outgoing card fades out at 150ms (`MintMotion.fast`)
- Replacement card fades in at 300ms (`MintMotion.standard`)
- No slide — pure crossfade only
- If no replacement card exists (empty state): fade to empty state widget, not to null
- Reduced-motion: skip animation entirely, swap instantly (check `MediaQuery.of(context).disableAnimations`)

### UXP-04: CustomTransitionPage Fade (Onboarding Journey)

**Applies to routes:** `/onboarding/intent`, `/coach`, `/onboarding/premier-eclairage`

**Spec (GoRouter `pageBuilder`):**
```dart
pageBuilder: (context, state) => CustomTransitionPage(
  key: state.pageKey,
  child: const TargetScreen(),
  transitionDuration: MintMotion.page,           // 350ms
  reverseTransitionDuration: MintMotion.standard, // 300ms
  transitionsBuilder: (context, animation, secondaryAnimation, child) {
    return FadeTransition(
      opacity: CurvedAnimation(
        parent: animation,
        curve: MintMotion.curveEnter,   // Curves.easeOutQuart
      ),
      child: child,
    );
  },
),
```

**Behaviour contract:**
- Forward (push): 350ms fade-in (`MintMotion.page`, `curveEnter`)
- Reverse (pop): 300ms fade-out (`MintMotion.standard`, `curveExit`)
- No slide, no scale — pure opacity transition
- All other routes in the app retain default Material slide behaviour
- Reduced-motion: replace FadeTransition with identity (opacity 1.0 always)

## ConfidenceScore Card Spec (UXP-02)

**Location:** `MintHomeScreen` — below `FinancialPlanCard`, above coach input bar.

**Visibility rule:** Always visible when `CoachProfile` has been created. Never hidden in settings.

**Layout:**
```
[MintSurface — tone: porcelaine, padding: 16px all]
  Row:
    Column (flex):
      Text "Ta précision de projection"  ← bodySmall, textSecondary
      SizedBox(height: 4)
      ConfidenceBar(score: score, showLabel: false, enrichmentActions: [topAction])
    SizedBox(width: 12)
    Column:
      Text "XX%"  ← bodySmall weight 700, semantic color
      Text zoneLabel  ← micro, semantic color
```

**Best improvement action display:**
- Show exactly 1 enrichment action (the first item from `EnhancedConfidence.enrichmentPrompts`)
- Label: the `enrichmentPrompts[0].label` string from the confidence scorer
- Tap: navigates to `data_block_enrichment` or the relevant input screen
- If no enrichment action available (score >= 95): show "Ta projection est très précise" in `bodySmall, textSecondary` instead

**Copywriting (French, informal "tu"):**
- Card header: "Ta précision de projection"
- Score label >= 70: "Bonne estimation"
- Score label >= 40: "Estimation large"
- Score label < 40: "On devine beaucoup"
- CTA prefix: "Pour aller plus loin\u00a0: [action label]"
- Full precision state: "Ta projection est très précise\u00a0— rien à ajouter pour l'instant."

## Explorer Hub ReadinessGate States (UXP-03)

**Driver:** `ReadinessGate.check(entry, profile)` for each hub's representative screen entry.

**Visual state mapping:**

| `ReadinessLevel` | Opacity | Icon tint | Indicator | Label overlay | Tap |
|-----------------|---------|-----------|-----------|---------------|-----|
| `ready` | 1.0 | normal (`MintColors.primary`) | none | none | active → `context.push(route)` |
| `partial` | 0.85 | normal | 8px `MintColors.warning` dot, top-right of icon | none | active → opens with partial banner |
| `blocked` | 0.55 | `MintColors.textMuted` | none | "Complète ton profil" (`bodySmall`, `textMuted`) | disabled (shows enrichment bottom sheet on tap) |

**Indicator dot (partial state):**
- 8px diameter circle
- Color: `MintColors.warning` (`#D97706`)
- Positioned: `Positioned(top: 0, right: 0)` inside a `Stack` wrapping the hub icon container
- No border, no shadow

**Blocked state bottom sheet (on tap):**
- Title: "Il te manque quelques infos"
- Body: Lists missing critical fields in plain French (e.g., "Ton salaire" for `salaireBrut`)
- CTA: "Compléter mon profil" → navigates to `data_block_enrichment`
- Sheet height: `DraggableScrollableSheet` initialChildSize 0.4, minChildSize 0.3, maxChildSize 0.7

**ReadinessGate hub mapping (which ScreenEntry to check per hub):**
- Retraite → check `salaireBrut`, `age`, `canton` (critical)
- Fiscalite → check `salaireBrut`, `canton` (critical)
- Logement → check `salaireBrut` (critical)
- Famille → check `civilStatus` (non-critical — always partial or ready)
- Travail → check `employmentStatus` (non-critical)
- Patrimoine → check `epargne` (non-critical)
- Sante → no required fields → always `ready`

## Accessibility Contract

- All animated widgets check `MediaQuery.of(context).disableAnimations` — if true, skip all transitions (pure instant swap)
- `ConfidenceBar` already has `Semantics(label: ...)` — `ConfidenceScoreCard` must preserve this
- Explorer hub cards: blocked state must expose `Semantics(button: true, label: 'Hub [name] — complète ton profil pour y accéder')`
- Explorer partial indicator dot: `Semantics(label: 'Données partielles')` on the dot widget
- CustomTransitionPage: no accessibility impact — fade does not affect focus order
- Touch targets: all hub cards minimum 44px height (existing `_ExploreHubCard` satisfies this)
