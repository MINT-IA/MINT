## Decisions Made

- **ReadinessGate inline implementation**: The plan referenced `ReadinessGate` and `ScreenEntry` services that don't exist in the codebase. Implemented equivalent logic directly in ExploreTab using static field requirements map and CoachProfile property reads. This is architecturally equivalent and avoids creating new service files not part of the plan scope.
- **canton 'VD' as default**: VD is the default canton value in wizard; treating it as "not set" is consistent with birthYear 1990 default treatment.
- **etatCivil always present**: CoachProfile always has a default etatCivil (celibataire) so famille hub never shows partial/blocked state — by design per UI-SPEC (famille uses optional not critical fields).
- **Snackbar for partial tooltip**: Used ScaffoldMessenger.showSnackBar instead of a custom Overlay for the long-press partial state tooltip — simpler, uses existing Material pattern, no memory leak risk.
- **/onboarding/premier-eclairage not found**: This route doesn't exist in the router. The premier eclairage content is surfaced as a widget on MintHomeScreen (via PremierEclairageCard from Plan 04-02), not as a separate navigation destination. Only 2 of the 3 planned routes were updated (as planned note said to skip if not found).
