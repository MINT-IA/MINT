## The 5 audit P0s closed

### Fix #1 — `lefthook-ci.yml` would have failed on first PR

- **Before:** used `lefthook validate` (not a lefthook 2.x subcommand) and `lefthook run commit-msg --file /tmp/msg.txt` (invalid flag). First PR would have gone red with cryptic error.
- **After:** `lefthook version` + `lefthook dump` for config check; direct `python3 tools/checks/proof_of_read.py <path>` for per-commit validation; pinned install to `v2.1.6`; added `timeout-minutes: 10`.

### Fix #2 — bypass-audit.yml could only see voluntary signals

- **Before:** grep regex = `LEFTHOOK_BYPASS|\[bypass:`. Missed `LEFTHOOK=0`, `LEFTHOOK_EXCLUDE`, `LEFTHOOK_RUN_ONLY`, `core.hooksPath`, `--no-verify`.
- **After:** regex extended to cover all 7 known vectors. Also passes count/triage via env vars to `github-script` (closes P2 template-literal injection via untrusted commit subjects).

### Fix #3 — `except: pass` and `catch (ex) {}` both bypassed

- **Before:** Dart regex hardcoded `(e|_|err|error)` identifier alternation; Python regex `$`-anchored (forbade same-line body). Audit 05 demonstrated 12 bypasses including the canonical `except: pass` one-liner.
- **After:** Dart widened to any valid identifier `[a-zA-Z_][a-zA-Z0-9_]*` with optional `(, stack)` 2nd arg. Python added 3 new patterns covering one-liners, tuple catches, typed catches, with optional trailing comment. +12 new pytest cases (24/24 green).

### Fix #4 — proof_of_read was effectively cosmetic

Four closed:

1. **Case-insensitive Co-Authored-By**: `Co-authored-by:` (Git canonical lowercase 'a') was treated as human commit, fully bypassing GUARD-06 doctrine.
2. **Trailer-block restriction**: `Read:` anywhere in the commit message passed. Now only the LAST paragraph (Git trailer convention) is scanned.
3. **Path canonicalisation**: `Read: .planning/phases/../../etc/passwd` passed string prefix check. Now `resolve(strict=False)` + `relative_to(allowed_root)` double-verifies the prefix AFTER resolving `..` segments and symlinks.
4. **Directory path crash**: `IsADirectoryError` exception when `Read:` pointed to a dir. Explicit `is_file()` check with differentiated failure messages.

Self-compliance: Fix #3 widening now catches the legitimate `except` clauses in Fix #4's implementation (structured error return contract). Added `# lefthook-allow:bare-catch:` overrides on preceding lines — **dogfooding the override mechanism designed for exactly this case**. +5 new pytest cases (17/17 green).

### Fix #5 — "Not documented = not banned" loophole

- **Before:** CONTRIBUTING §3 only mentioned `--no-verify` as banned. The other 6 bypass vectors were undocumented.
- **After:** New §3.5 "Known bypass vectors (banned by convention)" with an 8-row table mapping each vector to its detection mechanism. Explicit operator discipline clause: "find yourself typing any of these → stop and (a) fix, (b) inline override, or (c) escalate. No fourth path."

## What is NOT in 34.1 (deferred to 34.2)

Per audits/00-SYNTHESIS.md Tier C priorities:

1. **accent_lint_fr false-positive rate** (64–100% on `prevoyance`, `regler`, `liberer`, `preter`, `realiser`, `reperer` stems). Needs context-aware matching — skip code identifiers, handle Unicode normalisation / Cyrillic lookalikes, conjugation expansion. Scope: ~half-day job.
2. **`no-chiffre-choc` 50.6s perf cliff** (blows the 5s budget on any commit touching `.py|.dart|.arb`). Needs diff-only migration mirroring GUARD-02's state-machine diff parser. Scope: ~2h rewrite.
3. **ReDoS in `_TEXT_CAPITALISED`** regex (10.7s on 50KB unterminated `Text('A…`). Tighten pattern.
4. **ARB plural/select nested placeholder names** (audit 05): `{count, plural, one {{foo}} other {{bar}}}` emits only `count`. Recursive walker needed.
5. **ARB ordering drift** ({name} {age} vs {age} {name}): currently order-insensitive set compare; needs ordered vector compare for placeholder names appearing multiple times.
6. **Accent stems Unicode normalisation** (NFC vs NFD, fullwidth, Cyrillic lookalikes, zero-width space injection).
7. **Post-rewrite hook** to re-fire proof_of_read after `git commit --amend` / `git rebase -i` / `git cherry-pick`.
8. **no_hardcoded_fr whitelist regression** — `Text('BONJOUR')` passes because 7+ char rule. Needs context-aware.

These are catalogued with severity in `audits/00-SYNTHESIS.md` §"Tier C — Phase 34.2 polish".
