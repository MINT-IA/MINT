## Interaction Contracts

### Check-in Conversation Flow

**Trigger:** Nudge tap or "Faire mon point du mois" CTA tap ‚Üí opens CoachChatScreen with check-in pre-loaded.

**Conversation pattern:**
1. Coach sends opening message immediately on screen open (no user input required to start)
2. Coach asks one contribution question at a time ‚Äî never lists all questions at once
3. User types amount ("500" or "j'ai vers√© 500") ‚Äî natural language entry, no form fields
4. Parser extracts CHF amount and creates MonthlyCheckIn entry
5. After each answer, coach asks next PlannedMonthlyContribution question
6. Coach sends summary message and saves ‚Äî PlanRealityCard updates inline if visible

**Input pattern:** Standard Flutter TextField with `TextInputType.number`, no custom slider, no stepper. User types freely. Amount extraction is done by parser, not by UI constraint.

**Cross-session memory injection:** Coach opening message includes "Le mois dernier tu avais vers√© X CHF" when prior check-in exists. This is injected via ConversationMemoryService before sending to LLM ‚Äî not displayed as a separate UI element.

### PlanRealityCard on Aujourd'hui

**Visibility rule:** Only rendered when `CoachProfile.checkIns.isNotEmpty`. When no check-in exists, `FirstCheckInCtaCard` renders in its place.

**Streak integration:** `StreakBadgeWidget` is placed in the PlanRealityCard header row (right side, replacing empty space). It renders as `SizedBox.shrink()` when streak is 0 and totalCheckIns is 0 ‚Äî already implemented.

**Position on MintHomeScreen:** Section 2 ‚Äî after ChiffreVivant + PremierEclairageCard, before any other cards.

**Animation on check-in completion:** `AnimatedSwitcher` with crossfade duration 300ms when PlanRealityCard data updates after a check-in is saved. No hard refresh.

### Notification / Nudge

**No in-app UI screen for nudge itself.** OS local notification only, via NotificationService.scheduleNotification().

**Nudge tap behavior:** Opens CoachChatScreen via GoRouter deep link. Check-in mode is pre-activated. Coach immediately sends first check-in question.

**No nudge settings screen in Phase 5.** Notification consent is handled by existing NotificationService.
