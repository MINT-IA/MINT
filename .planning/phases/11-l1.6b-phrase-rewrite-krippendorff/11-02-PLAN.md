       - After last item: export JSON `{tester_id, seed, generated_at, ratings: [{item_id, rating, comment}]}` as downloadable file.
       - No network sends. No localStorage leak of previous runs (clear on start).
    3. `rater_ui_README.md`: one-page tester-facing README explaining how to open the HTML file locally, how to rate, how to return the exported JSON.
    4. Include a `test_mode` query param that loads 3 dummy items for rehearsal.
    5. Manual smoke test the HTML locally (serve via `python3 -m http.server` from `tools/krippendorff/`).
