# Plan 53-04 — Intent Contract Three-Way Parity (PLAN)

**Status:** Ready (must-fix BEFORE Phase 53 close-out audit declares PASS)
**Companion CONTEXT:** `53-CONTEXT.md`
**Estimated effort:** 1-2 days

## Goal (one sentence)

Close the contract gap caught by the post-Phase-53 5-expert panel's Engineering Reviewer: my Plan 53-01 ScreenRegistry parity gate enforces parity between `app.dart` and `MintScreenRegistry`, but it leaves untouched the two upstream whitelists that gate the chat-routing path — meaning 26 of the 33 ROUTABLE entries I added are physically unreachable from chat because the LLM can't suggest them and the mobile validator drops them.

## Why this plan now

Per Expert 2 (post-Phase-53 panel synthesis at `.planning/decisions/2026-05-04-phase-54-target.md`):

> Phase 53-01's "26 ROUTABLE entries" is a **registry-only mirage** — the LLM cannot suggest those intents and the mobile route-validator drops them — so Phase 54 must close the **chat→screen contract** before any chat-vivant scene work, otherwise Plan 53-02's `_injectSequencePrompt` will keep emitting `route_to_screen{intent: rente_vs_capital_arbitrage}` that the mobile silently swallows as `SizedBox.shrink()`.

The contract is split across three locations that drift independently:

1. `services/backend/app/services/coach/coach_tools.py:108-133` — `ROUTE_TO_SCREEN_INTENT_TAGS` lists exactly **24 intent tags**. Injected into the LLM system prompt at `claude_coach_service.py:206`. Claude is **physically incapable** of suggesting any tag not in this list.
2. `apps/mobile/lib/services/coach/tool_call_parser.dart:64-134` — `validRoutes` Set hardcodes **70 routes**. The validator at `widget_renderer.dart:110-117` returns `SizedBox.shrink()` for unknown routes.
3. `apps/mobile/lib/services/navigation/screen_registry.dart` — `MintScreenRegistry` (138 entries post-Plan 53-01).

Without single-source-of-truth, Plan 53-01's parity claim is misleading.

**Pre-existing test fragility** (Expert 2 evidence): `services/backend/tests/test_e2e_coach_pipeline.py:390` asserts `"life_event_divorce" in ROUTE_TO_SCREEN_INTENT_TAGS` — but the registry uses `life_event_divorce_v2`. Test passes; user-visible flow is dead. This plan also catches that class.

## Tasks

### T-53-04-01 — Build `MintScreenRegistry` JSON exporter (Dart side)

**Files:**
- New `apps/mobile/lib/services/navigation/screen_registry_export.dart` — pure function exporting `MintScreenRegistry.entries` to JSON shape `[{intentTag, route, preferFromChat}, ...]`
- New `apps/mobile/tools/dump_screen_registry.dart` — CLI entry-point that runs the exporter and writes to stdout (or to `tools/contracts/screen_registry.json` when `--write` is passed)

**Behavior:**

- CLI: `dart run apps/mobile/tools/dump_screen_registry.dart --write` regenerates `tools/contracts/screen_registry.json`.

**Test:** unit test asserting JSON shape stability + entry count == `MintScreenRegistry.entries.length`.

### T-53-04-02 — Codegen `validRoutes` (mobile side)

**Files:**
- `apps/mobile/lib/services/coach/tool_call_parser.dart` — replace the hand-maintained `validRoutes` Set with one read from a generated `apps/mobile/lib/services/coach/_valid_routes_generated.dart` (which is created by the dump CLI: filter `entries` where `preferFromChat == true`, project to `route`, dedupe, write as a `const Set<String>` literal).

**Behavior:**
- `tool_call_parser.dart` imports the generated set; no manual list maintenance.
- A drift CI gate fails if the generated set is out of sync with the registry.

**Test:** assert generated file matches a fixture-based regeneration.

## Risks + mitigations

| Risk | Mitigation |
|---|---|
| Codegen ordering: backend regen needs Dart export to run first; bootstrap is awkward | Commit `screen_registry.json` to the repo; backend regen reads the JSON not the Dart source. Dart export is the only step that needs `flutter` toolchain. CI runs Dart export → commits change → Python regen reads. |
| Some of the 70 hardcoded routes in `validRoutes` (`tool_call_parser.dart`) don't have `MintScreenRegistry` entries (legitimate cases like Phase 32 admin-only or anonymous wedge routes) | These should be in `_NOT_CHAT_ROUTABLE` per the screen registry registry-presence pattern (Plan 53-01); add explicit allow-list in the three-way parity lint matching the screen-registry KNOWN-MISSES doc. |
| LLM prompt regeneration: changing `ROUTE_TO_SCREEN_INTENT_TAGS` cascades into the system prompt that ships to Claude | Verify by snapshotting the system prompt before/after; assert the new prompt contains all 26 new intent tags and no removed ones. |
| Pre-existing test `test_e2e_coach_pipeline.py:390` will break when we replace the literal | Update the test in same PR (T-53-04-05). |
| Backend codegen runs at import time vs. build time — could slow startup | Generated file is a static set literal; same import cost as today. |
