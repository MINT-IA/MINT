<objective>
Fix the broken anonymous-to-authenticated conversation migration path. Anonymous messages currently live only in widget state (_messages list). When AuthGateBottomSheet navigates to /auth/register, AnonymousChatScreen is disposed and messages are lost. The auth_provider migration finds nothing to migrate.

Purpose: ANON-04 requires zero message loss on auth conversion. The migration infrastructure exists and works (conversation_store.migrateAnonymousToUser, auth_provider._migrateLocalDataIfNeeded), but the data never reaches SharedPreferences because messages are only saved in the unreachable _onAuthenticated callback.

Output: Anonymous messages persisted eagerly to SharedPreferences after each coach response. auth_provider migration path works regardless of how the user reaches the register screen.
</objective>
