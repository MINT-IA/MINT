### Pattern 1: FastMCP tool registration

**What:** Decorator-based tool exposure. Pydantic type hints become JSON Schema automatically.
**When to use:** Default for all 4 tools in this phase.
**Example:**

```python
# Source: https://pypi.org/project/mcp/ — official quickstart (FastMCP)
from mcp.server.fastmcp import FastMCP
from pydantic import BaseModel, Field
from typing import Literal

mcp = FastMCP("mint-tools")  # server name visible to Claude Code

TOOL_VERSION = "30.7.0"
Category = Literal["pillar3a", "lpp", "avs", "mortgage", "tax"]

class ConstantEntry(BaseModel):
    key: str
    value: float
    unit: str
    source: str  # e.g. "OPP3 art. 7 al. 1"

class GetConstantsResult(BaseModel):
    version: str = Field(default=TOOL_VERSION)
    category: str
    constants: list[ConstantEntry]

@mcp.tool()
def get_swiss_constants(category: Category) -> GetConstantsResult:
    """Return Swiss regulatory constants for a given category.

    Args:
        category: One of pillar3a, lpp, avs, mortgage, tax.
    """
    from app.services.regulatory.registry import RegulatoryRegistry
    params = RegulatoryRegistry.instance().get_all(category=category)
    return GetConstantsResult(
        category=category,
        constants=[
            ConstantEntry(
                key=p.key,
                value=p.value,
                unit=p.unit,
                source=p.source_title or "",
            )
            for p in params
        ],
    )

if __name__ == "__main__":
    mcp.run(transport="stdio")
```

**What:** Call existing `tools/checks/*.py` scripts as subprocesses, parse exit code + stderr.

```python
# Source: synthesised from existing tools/checks/* patterns (argparse + exit codes)
import subprocess, shutil
from pathlib import Path

ARB_PARITY_SCRIPT = Path("tools/checks/arb_parity.py")

def validate_arb_parity() -> dict:
    if not ARB_PARITY_SCRIPT.exists():
        return {
            "version": "30.7.0",
            "status": "lint_not_available",
            "reason": "Phase 34 GUARD-05 (arb_parity.py) not yet shipped",
            "script_expected_at": str(ARB_PARITY_SCRIPT),
        }
    result = subprocess.run(
        ["python3", str(ARB_PARITY_SCRIPT)],
        capture_output=True,
        text=True,
        timeout=30,
    )
    return {
        "version": "30.7.0",
        "status": "ok" if result.returncode == 0 else "drift_detected",
        "exit_code": result.returncode,
        "stdout": result.stdout[-4000:],  # truncate defensively
        "stderr": result.stderr[-4000:],
    }
```

### Anti-Patterns to Avoid

- **`print()` in tool handlers** — writes to stdout, which is the JSON-RPC channel. Corrupts MCP messages. Use `import sys; print(..., file=sys.stderr)` exclusively. `[CITED: code.claude.com/docs/en/mcp — "never use console.log() / print()"]`
- **Importing FastAPI or SQLAlchemy transitively** — slow startup, brittle for a dev-tool. Verify import graph: `RegulatoryRegistry` + `ComplianceGuard` + `HallucinationDetector` are all pure-Python (verified `[Read]` against files). Do not accidentally import anything from `services/backend/app/main.py`, `app/api/*`, or `app/models/*` that pulls SQLAlchemy.
- **Stateful caching between invocations** — contradicts the locked "stateless" decision. The stdio process is long-lived but each tool call must read sources from disk/registry. The `RegulatoryRegistry` singleton is acceptable (it's memoised module-level state, not cross-invocation cache).
- **Hardcoding tool output schema in CLAUDE.md or docs** — the schema lives in the Pydantic model. If the planner wants a CONTRACT.md, point it to the FastMCP-generated JSON Schema, not a hand-written duplicate.
- **Shelling out to `python3`** (without `.11`) inside subprocess calls — host default is 3.9.6. Use `sys.executable` or explicit `python3.11`.

## Common Pitfalls

### Pitfall 1: `print()` or `logging.basicConfig(stream=sys.stdout)` inside tools
**What goes wrong:** Any write to stdout from inside a tool handler corrupts the JSON-RPC message stream. Claude Code reports `Connection closed` or `malformed JSON` and the server dies.
**Why it happens:** Natural Python reflex. `ComplianceGuard` uses `logging` — if its default handler goes to stdout, we inherit that.
**How to avoid:** At `server.py` top: `import logging, sys; logging.basicConfig(stream=sys.stderr, level=logging.WARNING)`. Audit `ComplianceGuard` imports — it uses `logger = logging.getLogger(__name__)` which by default writes to stderr when no root handler is set, but the backend FastAPI config may reconfigure root. Isolate with an explicit handler. `[CITED: code.claude.com/docs/en/mcp — "never console.log/print"]`
**Warning signs:** First J0 session shows `Connection closed` with no context. Or `mcp` shows `Invalid JSON-RPC message`. Check `logs/` or stderr capture.

### Pitfall 2: Python 3.10+ import fails on host default `python3`
**What goes wrong:** Operator runs `python3 tools/mcp/mint-tools/server.py` manually (or `.mcp.json` forgets the `.11` suffix), hits `SyntaxError: invalid syntax` on an f-string or `ImportError: no module named 'mcp'`.
**Why it happens:** `/usr/bin/python3` on macOS Tahoe and our host both resolve to 3.9.6. `mcp` SDK requires ≥3.10.
**How to avoid:** Hardcode `python3.11` in `.mcp.json`. README includes `python3.11 --version` sanity check in the "Install" section. Wave 0 adds a `tools/mcp/mint-tools/tests/test_interpreter_version.py` that asserts `sys.version_info >= (3, 10)` as its first line — fails loud if run with the wrong interpreter.
**Warning signs:** First import of `mcp.server.fastmcp` raises `ModuleNotFoundError` despite `pip list` showing `mcp`.

### Pitfall 3: `PYTHONPATH` missing → `app.*` imports fail
**What goes wrong:** Tool-01 crashes with `ModuleNotFoundError: No module named 'app'` because `services/backend` is not on sys.path when the MCP process is spawned from repo root.
**Why it happens:** Claude Code invokes `command` from the workspace root, not from `services/backend/`. Python only auto-adds the script's own directory to `sys.path`.
**How to avoid:** `.mcp.json` `env.PYTHONPATH` = `${workspaceFolder}/services/backend:${workspaceFolder}/tools/mcp/mint-tools`. Plus `server.py` does a defensive prepend:
```python
import sys, pathlib
REPO_ROOT = pathlib.Path(__file__).resolve().parents[3]
sys.path.insert(0, str(REPO_ROOT / "services" / "backend"))
```

### Pitfall 4: `ComplianceGuard.__init__` side effects (HallucinationDetector instantiation)
**What goes wrong:** `ComplianceGuard()` in `__init__` creates `HallucinationDetector()` which may load a tokenizer, regex pack, or FinancialCore reference — slow startup, possible network call.
**Why it happens:** `compliance_guard.py:352-353` calls `HallucinationDetector()` unconditionally.
**How to avoid:** Instantiate `ComplianceGuard` once at module scope in `tools/banned_terms.py`, not per-invocation. Time the first call; if cold start > 500 ms, switch to a narrow wrapper that only calls `_check_banned_terms` and `_sanitize_banned_terms` (both are classmethod-able — they only need `_get_banned_patterns()` which is a classmethod + the TERM_REPLACEMENTS dict). `[VERIFIED: Read compliance_guard.py:611-654]`
**Warning signs:** Tool-02 latency > 1 s on first call even for short text.

### Pitfall 5: TOOL-03 graceful fallback silently turns into ship-blocker
**What goes wrong:** Agents get `{status: "lint_not_available"}` from TOOL-03 and treat the report as "parity OK" because the payload is JSON and doesn't raise.
**Why it happens:** LLMs pattern-match on JSON success envelopes.
**How to avoid:** The fallback payload uses a distinct `status` value (`"lint_not_available"`, not `"ok"`) AND the tool docstring explicitly warns: "If status is `lint_not_available`, DO NOT conclude ARB parity is OK — invoke the flutter gen-l10n step manually and grep ARB files for keyset equality." The integration test asserts the exact string `lint_not_available` when `arb_parity.py` is absent.
**Warning signs:** First agent post-30.7 claims "ARB parity verified via tool" in a commit message despite Phase 34 not being shipped. Correct with prompt hint.

### Pitfall 6: Windows / WSL path handling
**What goes wrong:** Phase-34 `arb_parity.py` subprocess call fails on Windows with path encoding issues.
**Why it happens:** Not a concern today — MINT dev happens on macOS Tahoe per MEMORY.md. But CI runs on Ubuntu.
**How to avoid:** Use `pathlib.Path` exclusively. Subprocess args as list (not shell string). CI job runs the integration test in Python 3.11 Ubuntu matrix.
**Warning signs:** Only surfaces in CI. Unlikely to trip before Phase 34 wires the actual lint.

### Pitfall 7: `.mcp.json` approval prompt silently rejected
**What goes wrong:** Claude Code prompts "Allow mint-tools MCP server?" on first load. If Julien dismisses or closes the prompt, the server never starts, tools are invisible, and agents silently fall back to CLAUDE.md-style knowledge. But CLAUDE.md has been TRIMMED. Result: agents operate with incomplete context, drift spikes.
**Why it happens:** Project-scope MCP requires per-user approval for security. `[CITED: code.claude.com/docs/en/mcp §Project scope]`
**How to avoid:** SUMMARY.md has a "First-run checklist" with screenshot of the expected prompt and explicit "Click Accept". J0 validation gate measures "4 tools invoked ≥1× per session" — if Session 1 shows 0 invocations, the approval was dismissed; re-run and require explicit confirmation.
**Warning signs:** Dashboard (Phase 30.5) shows tool invocation count = 0 on Session 1.

### Pitfall 8: Token budget measurement is imprecise — -30% target unverifiable
**What goes wrong:** We claim "-30% CLAUDE.md tokens" but tokens ≠ bytes ≠ words. Without Claude's exact tokenizer we can only estimate.
**Why it happens:** CLAUDE.md is auto-loaded. Claude's tokenizer (SentencePiece variant) has a ~4 chars/token average for French/English mixed text, but variance ±20%.
**How to avoid:** Measure THREE numbers, commit all three in SUMMARY.md:
  1. **Lines:** `wc -l CLAUDE.md` — 121 today → target ≤ 85 (-30%).
  2. **Bytes:** `wc -c CLAUDE.md` — 8019 today → target ≤ 5613 (-30%).
  3. **Estimated tokens:** `python3 -c "print(int($(wc -c < CLAUDE.md) / 4))"` — 2005 today → target ≤ 1404 (-30%).
  Any two of the three falling ≥30% is sufficient; by-the-letter the ROADMAP says "core tokens -30%" (criterion #4), so the estimated-tokens number is the primary signal.
**Warning signs:** Measurement script returns -20% — trim more. Or returns -45% — we over-trimmed, check that TOP/BOTTOM top-3 critical rules are still present.

### Pitfall 9: Atomic CLAUDE.md commit violates `claude_md_bracket.py`
**What goes wrong:** After removing §5 + §6, the TOP/BOTTOM bracketing rule (Liu 2024) breaks because rule #1 (banned terms) now has a shorter "Full list →" redirect instead of the full list. The `tools/checks/claude_md_bracket.py` lint may flag this.
**Why it happens:** `claude_md_bracket.py` enforces structural invariants on CLAUDE.md. We don't yet know which invariants.
**How to avoid:** Wave 0 includes a dry-run of `python3 tools/checks/claude_md_bracket.py` against a draft post-trim CLAUDE.md to find violations. Adjust trim to keep structure intact. **Required step, not optional.**
**Warning signs:** lefthook pre-commit fires on the atomic trim commit.

### Pitfall 10: FastMCP signature changes between 1.x and 2.x
**What goes wrong:** Between Wave 0 (install) and Wave 2 (integration), `mcp` SDK releases a breaking change (e.g. `@mcp.tool()` → `@mcp.register_tool()`).
**Why it happens:** `mcp` is on v1.27 as of 2026-04-02, minor version every ~2 weeks. `[VERIFIED: PyPI]`
**How to avoid:** `requirements.txt` pins `mcp>=1.9,<2.0` (hard upper bound on major). Re-run the pin during Wave 0; commit the exact version that passed integration tests. CI smoke runs `pytest` on every push so any SDK drift fails loud.
**Warning signs:** `pip install -U mcp` in a week suddenly breaks `server.py`.

### Tool 2: `check_banned_terms(text)` — wraps `ComplianceGuard`

```python
# Source: services/backend/app/services/coach/compliance_guard.py:611 (_check_banned_terms)
# + :637 (_sanitize_banned_terms) + :170 (TERM_REPLACEMENTS)
from mcp.server.fastmcp import FastMCP
from pydantic import BaseModel

# Module-scope instantiation — avoid per-call init cost
from app.services.coach.compliance_guard import ComplianceGuard
_guard = ComplianceGuard()

class BannedTermHit(BaseModel):
    term: str
    suggestion: str

class BannedTermsResult(BaseModel):
    version: str = "30.7.0"
    banned_found: list[str]         # raw terms as ComplianceGuard returns them
    suggestions: list[BannedTermHit]  # term + canonical replacement
    sanitized_text: str              # ComplianceGuard's rewrite output

@mcp.tool()
def check_banned_terms(text: str) -> BannedTermsResult:
    """Detect LSFin-banned terms in a proposed string and suggest replacements.

    Uses the production ComplianceGuard pipeline (Layer 1 only — banned terms).
    Does NOT run the full 5-layer validation. Safe to invoke on drafts,
    variable names, CLAUDE.md edits, etc.

    Returns:
      - banned_found: list of banned terms present (may include plurals, gerunds,
        conditionals, context-aware "certain" family)
      - suggestions: {term, replacement} pairs from the canonical TERM_REPLACEMENTS map
      - sanitized_text: the input with compliant replacements applied
    """
    found = _guard._check_banned_terms(text)  # classmethod-like, safe to call
    sanitized = _guard._sanitize_banned_terms(text)
    suggestions = [
        BannedTermHit(term=t, suggestion=ComplianceGuard.TERM_REPLACEMENTS.get(t, ""))
        for t in found
    ]
    return BannedTermsResult(
        banned_found=found,
        suggestions=suggestions,
        sanitized_text=sanitized,
    )
```

### Tool 3: `validate_arb_parity()` — graceful subprocess wrap

```python
# Source: pattern synthesised from tools/checks/route_registry_parity.py (exit codes + argparse)

import subprocess, sys
from pathlib import Path
from mcp.server.fastmcp import FastMCP
from pydantic import BaseModel

REPO_ROOT = Path(__file__).resolve().parents[3]
ARB_PARITY_SCRIPT = REPO_ROOT / "tools" / "checks" / "arb_parity.py"

class ArbParityResult(BaseModel):
    version: str = "30.7.0"
    status: str  # "ok" | "drift_detected" | "lint_not_available"
    exit_code: int | None = None
    stdout_tail: str = ""
    stderr_tail: str = ""
    reason: str = ""

@mcp.tool()
def validate_arb_parity() -> ArbParityResult:
    """Run the ARB parity lint across the 6 locales (fr, en, de, es, it, pt).

    IMPORTANT: When status is 'lint_not_available', DO NOT conclude that ARB
    files are in sync. The Phase 34 lint is not yet shipped; invoke
    `flutter gen-l10n` manually and grep ARB files for keyset equality instead.
    """
    if not ARB_PARITY_SCRIPT.exists():
        return ArbParityResult(
            status="lint_not_available",
            reason=(
                f"tools/checks/arb_parity.py not found. "
                f"This is a Phase 34 GUARD-05 deliverable. "
                f"Until then, use `flutter gen-l10n` + manual keyset diff."
            ),
        )
    try:
        result = subprocess.run(
            [sys.executable, str(ARB_PARITY_SCRIPT)],
            capture_output=True, text=True, timeout=30,
            cwd=str(REPO_ROOT),
        )
    except subprocess.TimeoutExpired:
        return ArbParityResult(status="drift_detected", reason="lint timed out (30s)")
    return ArbParityResult(
        status="ok" if result.returncode == 0 else "drift_detected",
        exit_code=result.returncode,
        stdout_tail=result.stdout[-2000:],
        stderr_tail=result.stderr[-2000:],
    )
```

### Tool 4: `check_accent_patterns(text)` — wraps `accent_lint_fr.py`

```python
# Source: tools/checks/accent_lint_fr.py:28 (PATTERNS) + :57 (scan_file signature)
# scan_file takes a Path; we need a scan_text() equivalent — minimal refactor:
import re
from mcp.server.fastmcp import FastMCP
from pydantic import BaseModel

# Import the pattern list directly — single source of truth
import sys, pathlib
sys.path.insert(0, str(pathlib.Path(__file__).resolve().parents[3] / "tools" / "checks"))
import accent_lint_fr

class AccentHit(BaseModel):
    line: int
    snippet: str
    pattern: str
    suggestion: str

class AccentResult(BaseModel):
    version: str = "30.7.0"
    violations: list[AccentHit]
    clean: bool

@mcp.tool()
def check_accent_patterns(text: str) -> AccentResult:
    """Scan French text for ASCII-flattened words that should carry diacritics.

    Uses the PATTERNS list from tools/checks/accent_lint_fr.py (single source
    of truth — Phase 34 GUARD-04 extends it, and we inherit automatically).
    """
    hits: list[AccentHit] = []
    for lineno, line in enumerate(text.splitlines(), start=1):
        for pat, correct in accent_lint_fr.PATTERNS:
            if re.search(pat, line, re.IGNORECASE):
                hits.append(AccentHit(
                    line=lineno,
                    snippet=line.strip()[:140],
                    pattern=pat,
                    suggestion=correct,
                ))
    return AccentResult(violations=hits, clean=(not hits))
```

Note: This calls for a minor companion change in `tools/checks/accent_lint_fr.py` — exposing a `scan_text(text: str)` function alongside `scan_file(path: Path)` so the MCP tool can reuse it cleanly. The change is additive (no removal), backward-compatible with Phase 34.

### Wave 0: `measure_context_budget.py` — baseline + post-trim delta

```python
# Source: new; measurement instrument for ROADMAP success criterion #4 (-30% tokens)
"""Measure CLAUDE.md size pre/post Phase 30.7 trim.

Run once pre-trim (Wave 0) to capture baseline.
Run once post-trim (last wave) to compute delta.
Commit both outputs to SUMMARY.md.
"""
import sys, pathlib, subprocess

CLAUDE_MD = pathlib.Path(__file__).resolve().parents[3] / "CLAUDE.md"

def measure(path: pathlib.Path) -> dict:
    text = path.read_text(encoding="utf-8")
    lines = text.count("\n")
    bytes_ = len(text.encode("utf-8"))
    # Rough token estimate: ~4 chars/token for mixed FR/EN text (empirical heuristic).
    # Not tiktoken because Claude's tokenizer is proprietary; this is a proxy.
    est_tokens = bytes_ // 4
    return {"lines": lines, "bytes": bytes_, "est_tokens": est_tokens}

if __name__ == "__main__":
    m = measure(CLAUDE_MD)
    print(f"CLAUDE.md: {m['lines']} lines, {m['bytes']} bytes, ~{m['est_tokens']} tokens")
    print(f"  -30% targets: {int(m['lines']*0.7)} lines, {int(m['bytes']*0.7)} bytes, ~{int(m['est_tokens']*0.7)} tokens")
```

- Estimated tokens: ~2005

## Security Domain

Minimal surface — this is developer tooling, not user-facing runtime. Still:

### Applicable ASVS Categories

| ASVS Category | Applies | Standard Control |
|---------------|---------|-----------------|
| V2 Authentication | no | MCP stdio runs as local child process under user's own credentials. No auth model. |
| V3 Session Management | no | Each Claude Code session spawns its own MCP subprocess. No session state crossing users. |
| V4 Access Control | no | Tools are read-only. No write operations on the filesystem (except TOOL-03 reads ARB files via subprocess). |
| V5 Input Validation | **yes** | Pydantic v2 types validate every tool input. `Literal["pillar3a", ...]` rejects unknown categories. String inputs have no length cap in MCP protocol — add `Field(..., max_length=100_000)` on `text` params to prevent DoS via enormous payloads. |
| V6 Cryptography | no | No secrets handled. No signing. |
| V7 Error Handling | **yes** | Tool errors must return structured JSON (not raise). Stacktraces go to stderr, never stdout. Subprocess timeouts (TOOL-03) are enforced at 30s. |
| V8 Data Protection | no | No PII touched. ARB lint reads ARB files but doesn't exfiltrate content — only keyset-level reports. |
| V11 Business Logic | **yes (light)** | TOOL-03 fallback status MUST be visibly distinct from `ok` (pitfall #5). Integration test asserts the exact string. |
| V14 Configuration | **yes** | `.mcp.json` committed to git — contains no secrets (only PYTHONPATH + command path). Verify via grep pre-commit. |

### Known Threat Patterns for Python stdio MCP server

| Pattern | STRIDE | Standard Mitigation |
|---------|--------|---------------------|
| Malicious input crashes server (e.g. enormous string) | Denial of Service | Pydantic `max_length` on `text` inputs; subprocess timeouts 30s for TOOL-03 |
| `print()` / `logger.info()` corrupts JSON-RPC frame | Tampering (accidental) | Global `logging.basicConfig(stream=sys.stderr)` at `server.py` top; integration test `test_no_stdout_pollution` |
| Subprocess invocation with attacker-controlled args | Elevation of Privilege (hypothetical) | TOOL-03 subprocess args are hardcoded list; no user input flows into `subprocess.run` args |
| `PYTHONPATH` injection via `.mcp.json` tampering | Tampering | `.mcp.json` is committed + reviewed in PR; project-scope approval prompt on first load adds a user-consent gate `[CITED: code.claude.com/docs/en/mcp]` |
| Banned-term leak via TOOL-02 response | Information Disclosure (trivial — terms are already public) | Non-issue. TOOL-02 output IS the banned-term set. |
