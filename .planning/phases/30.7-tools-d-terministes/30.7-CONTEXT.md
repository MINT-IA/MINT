**Mode:** Expert auto-answers (full autonomous)

<domain>
## Phase Boundary

Transformer les constantes et lints permanents de CLAUDE.md en **MCP tools on-demand**. Les agents invoquent `get_swiss_constants()` / `check_banned_terms()` / `validate_arb_parity()` quand pertinent au lieu de traîner 800+ tokens de règles en contexte permanent.

**In scope:**

- Multi-MCP-server orchestration (single server, 3 tools)

<decisions>
## Implementation Decisions

### Area 1: MCP server runtime & packaging
- **Runtime: Python 3.11+** — consistency with `services/backend/`, `tools/checks/*.py`, reuse pytest fixtures. Node would split ecosystem.
- **Location: `tools/mcp/mint-tools/`** — new subdir, separate from Railway deploy path. Not installed in prod image.
- **Transport: MCP stdio** — standard for Claude Code local tools. No HTTP server needed.
- **Registration: `.mcp.json` at repo root + `.claude/settings.json` reference** — matches GSD pattern already used in `.claude/get-shit-done/`. Commit `.mcp.json`; `.claude/settings.local.json` ignored per gitignore.

### Area 2: Tool scope & contracts
- **`get_swiss_constants(topic: str)`** — topic ∈ {`lpp`, `avs`, `lifd`, `archetypes`, `banned_terms`, `accents`}. Returns versioned JSON pulled from `docs/AGENTS/swiss-brain.md` front-matter + ancillary sources. Single tool, topic-scoped response.
- **`check_banned_terms(text: str, strict: bool=False)`** — returns `[{term, offset, severity, suggested_replacement}]`. `strict=True` activates accent check (ASCII `e` → bug per CLAUDE.md §2). Wraps `tools/checks/accent_lint_fr.py` logic.
- **`validate_arb_parity(key: str=None)`** — if `key` given, returns 6-lang coverage for that key; else global report `{orphans, missing, divergent}`. Reads `apps/mobile/lib/l10n/*.arb` directly.
- **Stateless** — each invocation reads source-of-truth. No cache. Reproducible, cheap.
- **Versioning** — each tool returns `{version: "30.7.0", payload: ...}`. Version bump on schema change; agents can assert compatibility.

### Area 3: CLAUDE.md migration
- **Full banned-terms list (~20 terms in `docs/AGENTS/swiss-brain.md`) moves OUT of CLAUDE.md loaded context**, stays on disk. `get_swiss_constants("banned_terms")` is the on-demand route.
- **CLAUDE.md retains top-3 critical terms** (« garanti », « optimal », « meilleur ») + pointer to tool — Liu 2024 lost-in-the-middle keeps critical in prompt.
- **Accent rule stays in CLAUDE.md** — too frequent, always-on rule. `check_banned_terms(strict=True)` wraps it so agents validate on demand (e.g., before commit).
- **i18n rule stays in CLAUDE.md** — critical framework rule. `validate_arb_parity()` becomes the enforcer agents call before PR.
- **Target budget: CLAUDE.md auto-loaded portion ≤100 lines** (currently ~150). Measurement via existing `.claude/hooks/context-monitor.js` baseline, diff before/after.

### Area 4: Rollout, validation, kill-policy
- **Exposure: MCP server auto-starts via Claude Code `.mcp.json`** — tools discoverable via ToolSearch at runtime. No per-agent wiring.
- **Validation gate (J0):** 3 fresh Claude Code sessions each (a) invoke each tool ≥1× successfully, (b) show token delta on `.claude/hooks/context-monitor.js` baseline (target ≥-500 tokens in auto-loaded context), (c) `flutter analyze` + `pytest -q` remain green.
- **Kill-policy (v2.8 doctrine):** MCP server fails to start 2× OR any tool returns malformed JSON in real agent use → revert `.mcp.json` registration, fallback to full-text CLAUDE.md. Phase 30.7 code stays on disk for retry. Kill-switch: delete `.mcp.json` line (not code).
- **Ship criteria:** 3 tools live, `.mcp.json` committed, 1 pytest integration test per tool, CLAUDE.md trimmed + diff documented, SUMMARY lists token budget delta + kill-switch instructions.

### Claude's Discretion
- Exact file layout under `tools/mcp/mint-tools/` (prefer: `server.py`, `tools/{constants,banned_terms,arb_parity}.py`, `tests/`)
- Internal JSON schema per tool payload (follow MCP SDK conventions)
- Whether to use `anthropic-mcp` Python SDK or raw stdio — default to official SDK if available in April 2026
- Exact version string format — default `{major}.{minor}.{patch}` matching phase number

</decisions>

<specifics>
## Specific Ideas

- **Naming:** MCP server identifier `mint-tools` (not `mint-mcp`). Tool names use snake_case to match MCP convention.
- **Version field:** `"30.7.0"` — first deployment. Bump minor for schema additions (new topic in `get_swiss_constants`), major for breaking changes.
- **Banned terms source of truth:** single YAML/JSON file at `tools/mcp/mint-tools/data/banned_terms.yaml` rather than scraping `docs/AGENTS/swiss-brain.md` — prevents drift. `swiss-brain.md` becomes the human-readable surface; YAML is the machine surface.
- **ARB parity: match existing CI script output format.** If `.github/workflows/` already has an `arb_parity_check.py`, reuse its parsing.
- **Token budget measurement:** instrument via simple script — `tools/mcp/mint-tools/tests/measure_context_budget.py` — that runs before/after, prints delta. Commit baseline and post-phase numbers in SUMMARY.
- **Doctrine anchor:** « pas de façade sans câblage ». Every tool must have (a) callable unit test, (b) proof of at least 1 real agent invocation in J0 gate, (c) kill-switch documented. No tool ships without the triangle.

</specifics>
