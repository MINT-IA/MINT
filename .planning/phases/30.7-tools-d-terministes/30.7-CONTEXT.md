<decisions>
## Implementation Decisions

### Area 1: MCP server runtime & packaging
- **Runtime: Python 3.11 (hardcoded)** — host default `python3` is 3.9.6 but `mcp` SDK ≥1.9 requires ≥3.10. `python3.11` available via pyenv shim. `.mcp.json` MUST use `"command": "python3.11"` explicitly.
- **SDK: Official `mcp` Python package ≥1.9,<2.0** (currently 1.27.0 on PyPI, 2026-04-02). Use `FastMCP` decorator API + stdio transport. No separate `fastmcp` dep.
- **Location: `tools/mcp/mint-tools/`** — new subdir, separate from Railway deploy path. Dedicated venv (isolation > convenience).
- **Transport: MCP stdio** — standard for Claude Code local tools. No HTTP server.
- **Registration: `.mcp.json` at repo root** — project-scope config per code.claude.com/docs/en/mcp. `mcpServers.mint-tools` with `command: "python3.11"` + `args` + `env.PYTHONPATH`. Triggers first-run approval prompt (documented in SUMMARY).

### Area 2: Tool scope & contracts (4 tools, TOOL-01..04)
- **TOOL-01 `get_swiss_constants(category: str)`** — category ∈ {`pillar3a`, `lpp`, `avs`, `mortgage`, `tax`}. **Source: `services/backend/app/services/regulatory/registry.py`** singleton (`RegulatoryRegistry.instance().get_all(category=...)`). All 5 categories already supported. DO NOT duplicate. Versioned: constants 2025/2026.
- **TOOL-02 `check_banned_terms(text: str)`** — wraps `ComplianceGuard` at **`services/backend/app/services/coach/compliance_guard.py`** (corrected path). Uses `_check_banned_terms` + `_sanitize_banned_terms` + `TERM_REPLACEMENTS`. Layer 1 only (banned terms); Layer 2/2b deferred to v2.9. Returns `{banned_found: [...], suggestions: [...]}`.
- **TOOL-03 `validate_arb_parity()`** — wraps `tools/checks/arb_parity.py` (Phase 34 lint). Returns global ARB parity report across 6 languages. Agents invoke before i18n-touching PRs.
- **TOOL-04 `check_accent_patterns(text: str)`** — wraps `tools/checks/accent_lint_fr.py`. Returns accent violations (ASCII `e` where `é` expected, etc). Agents invoke on FR content before commit.
- **Stateless** — each invocation reads source-of-truth. No cache. Reproducible, cheap.
- **Versioning** — each tool returns `{version: "30.7.0", payload: ...}`. Version bump on schema change; agents can assert compatibility.
- **Dependency on Phase 34:** `arb_parity.py` is a Phase 34 deliverable. TOOL-03 has a fallback: if script missing at invocation time, returns `{status: "lint_not_available", reason: "phase 34 not shipped"}` rather than failing. Prevents 30.7 ship-block on 34 ordering.

### Area 3: CLAUDE.md migration (ROADMAP-mandated -30%)
- **§5 BUSINESS RULES (constants) — REMOVED from CLAUDE.md**, replaced by pointer to `get_swiss_constants(category)`. Constants live in `services/backend/app/constants/` (already source of truth, not duplicated).
- **§6 COMPLIANCE banned-terms list — REMOVED from CLAUDE.md**, replaced by `check_banned_terms(text)` on-demand. CLAUDE.md TOP/BOTTOM rule #1 retains top-3 critical terms (« garanti », « optimal », « meilleur ») per Liu 2024 lost-in-the-middle — DO NOT move these.
- **Accent rule (TOP/BOTTOM rule #2)** — stays in CLAUDE.md (high-frequency, always-on). `check_accent_patterns(text)` supplements for agent on-demand validation before commit.
- **i18n rule (TOP/BOTTOM rule #5)** — stays in CLAUDE.md. `validate_arb_parity()` becomes the enforcer agents call before i18n-touching PRs.
- **Target budget:** CLAUDE.md auto-loaded portion **-30% tokens** (ROADMAP success criterion). Measurement via existing `.claude/hooks/context-monitor.js` baseline, diff before/after. Pre-baseline captured in SUMMARY.md.

### Area 4: Rollout, validation, kill-policy
- **Exposure: MCP server auto-starts via Claude Code `.mcp.json`** — tools discoverable via ToolSearch at runtime. No per-agent wiring.
- **Validation gate (J0):** 3 fresh Claude Code sessions each (a) invoke each of the 4 tools ≥1× successfully, (b) show token delta on `.claude/hooks/context-monitor.js` baseline (target ≥-30% per ROADMAP, min acceptable -20%), (c) `flutter analyze` + `pytest -q` remain green. Dashboard from Phase 30.5 measures invocation frequency (success criterion 4: tools invoked ≥1×/session on relevant tasks).
- **Kill-policy (v2.8 doctrine):** MCP server fails to start 2× OR any tool returns malformed JSON in real agent use → revert `.mcp.json` registration, fallback to full-text CLAUDE.md (revert CLAUDE.md §5/§6 from git). Phase 30.7 code stays on disk for retry. Kill-switch: delete `.mcp.json` line + `git revert` CLAUDE.md trim commit.
- **Ship criteria:** 4 tools live, `.mcp.json` committed, 1 pytest integration test per tool, CLAUDE.md trimmed ≥30% + diff documented, SUMMARY lists token budget delta + kill-switch instructions.

## Specific Ideas

- **Naming:** MCP server identifier `mint-tools` (not `mint-mcp`). Tool names use snake_case to match MCP convention.
- **Version field:** `"30.7.0"` — first deployment. Bump minor for schema additions (new topic in `get_swiss_constants`), major for breaking changes.
- **Banned terms source of truth:** MCP tool imports `from app.services.coach.compliance_guard import ComplianceGuard` (NO duplication).
- **Swiss constants source of truth:** MCP tool imports `RegulatoryRegistry` singleton (NO duplication). 2026/2025 constants already present.
- **CLAUDE.md baseline (measured):** 121 lines, 8019 bytes, ~2005 tokens. -30% target = ≤85 lines, ≤5613 bytes, ~1403 tokens. -20% floor = ≤96 lines. Removing §5 constants + §6 NEVER #5 body reclaims ~40-50% — target feasible.
- **Phase 34 coupling:** `arb_parity.py` is a Phase 34 deliverable. TOOL-03 has graceful fallback `{status: "lint_not_available"}` if script missing. Phase 30.7 ships standalone; full TOOL-03 coverage activates when Phase 34 lands.
- **Token budget measurement:** instrument via simple script — `tools/mcp/mint-tools/tests/measure_context_budget.py` — that reads CLAUDE.md pre/post trim, prints delta. Commit baseline and post-phase numbers in SUMMARY.
- **CLAUDE.md edit safety:** single atomic commit for §5+§6 removal (not piecemeal). Precommit-check: `claude_md_bracket.py` must still pass post-edit.
- **Doctrine anchor:** « pas de façade sans câblage ». Every tool must have (a) callable unit test, (b) proof of at least 1 real agent invocation in J0 gate, (c) kill-switch documented. No tool ships without the triangle.
