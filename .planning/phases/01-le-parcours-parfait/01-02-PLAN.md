    - POST /api/v1/auth/magic-link/send with {email} creates a 32-byte token via secrets.token_urlsafe(), stores hash in DB with 15-min expiry, sends email
    - POST /api/v1/auth/magic-link/verify with {token} validates token, returns JWT access token, invalidates token (single-use)
    - Rate limit: 5 sends per minute per email (via slowapi)
    - Expired token returns 401
    - Already-used token returns 401
    - Invalid token returns 401
    - If email not registered, auto-create user account (frictionless onboarding)

    - Email sending via Resend API (with graceful fallback logging if RESEND_API_KEY not set)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> backend auth endpoints | Untrusted email input, token input |
| email channel | Magic link token transmitted via email (not encrypted channel) |
| deep link -> app | Token received from URL could be spoofed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02 | Tampering | /auth/magic-link/verify | mitigate | Rate limit 5/min, token >= 32 bytes urlsafe, single-use invalidation, SHA-256 hash stored (never plaintext) |
| T-01-03 | Information Disclosure | magic link email | accept | Token expires in 15 min, HTTPS only for verify endpoint, single-use. Email interception is inherent to magic link pattern -- accepted industry-wide. |
| T-01-04 | Spoofing | /auth/magic-link/send | mitigate | Rate limit 5/min per email prevents enumeration and spam. No confirmation whether email exists (same response for known/unknown). |
| T-01-05 | Denial of Service | /auth/magic-link/send | mitigate | slowapi rate limiting at 5/minute prevents abuse. Email service has its own rate limits. |
| T-01-06 | Elevation of Privilege | auto-create user on magic link | accept | Auto-create is intentional (frictionless onboarding). New users get default permissions only. No admin escalation path. |
</threat_model>
