## Widget Placement Contract

**Rule (D-06):** Widgets render inline, below the coach message text within the same message bubble column. They do not appear as floating overlays or separate chat turns.

**Layout within a coach message:**
1. Coach avatar (24px dot) + message text bubble — `MintColors.craie` background
2. Immediately below text (8px gap): rich widgets in order of `richToolCalls` list
3. Each widget wrapped in `ChatCardEntrance` — entrance animation on first render
4. Multiple widgets in one turn: staggered 80ms apart (established `ChatCardEntrance` delay pattern)

**RouteSuggestionCard layout:**
- Background: `MintColors.surface` (#F5F5F7), border-radius 12px
- Padding: `MintSpacing.md` (16px) all sides
- Context message: `MintTextStyles.bodyLarge()` — `MintColors.textSecondary`
- Warning banner (isPartial=true): amber background (`MintColors.warningBg`), `MintTextStyles.bodySmall()`, text from `routeSuggestionPartialWarning` ARB key
- CTA button: full-width, `MintColors.accent` fill, white text, `MintTextStyles.titleMedium()`, min height 44px, text from `routeSuggestionCta` ARB key ("Ouvrir")
- Blocked state: muted card, disabled CTA, text from `routeSuggestionBlocked` ARB key

**Unknown tool names:** Silent — no visible error widget. Debug log only. Message text renders normally. (Per D-08.)

## Motion Contract

All values sourced from `MintMotion` in `apps/mobile/lib/theme/mint_motion.dart`.

| Interaction | Duration | Curve | Notes |
|-------------|----------|-------|-------|
| Widget card entrance (slide+fade) | 600ms (MintMotion.slow) | easeOutQuart (MintMotion.curveEnter) | Via ChatCardEntrance — already implemented |
| Stagger offset between multiple cards | +80ms per card | — | Delay parameter on ChatCardEntrance |
| CTA button tap feedback | 150ms (MintMotion.fast) | easeOutCubic (MintMotion.curveStandard) | Standard opacity press response |

**Motion rules (from MINT_UX_GRAAL_MASTERPLAN §6):**
- No bounce, no overshoot, no spring on chat widgets
- No confetti or celebratory particle effects
- Apparition douce — calm, not urgent

## Copywriting Contract

All user-facing strings go through `AppLocalizations` (ARB files). French is the template locale (app_fr.arb).

| Element | ARB Key | French Copy | Notes |
|---------|---------|-------------|-------|
| Route suggestion CTA | `routeSuggestionCta` | "Ouvrir" | Already exists in all 6 locales |
| Route suggestion warning | `routeSuggestionPartialWarning` | "Estimation — données incomplètes" | Already exists in all 6 locales |
| Route blocked | `routeSuggestionBlocked` | "Il me manque des infos pour t'y emmener" | Already exists in all 6 locales |
| Score gauge title | `scoreGaugeTitle` | (existing) | Already exists |
| Score gauge disclaimer | `scoreGaugeDisclaimer` | (existing) | Already exists |
| Empty state (no profile) | `coachEmptyStateMessage` | "Pas encore de profil. Trois questions, et on discute." | Already exists |

**New ARB keys required for Phase 2** (add to all 6 locales before merging):

| ARB Key | French Copy | Context |
|---------|-------------|---------|
| `toolDispatchErrorBanner` | "Une réponse n'a pas pu s'afficher" | Shown only in debug/dev builds when a tool parse fails — not shown in production (D-08 silent fail) |

**Copywriting rules (from MINT voice system):**
- Informal "tu" throughout
- Non-breaking space (`\u00a0`) before `!`, `?`, `:`, `;`, `%`
- No banned terms: "garanti", "optimal", "meilleur", "parfait" (absolute)
- No clichés ("Explorer", "Découvrir") as CTA labels — use specific verb+noun ("Ouvrir le simulateur")
- Voice: calme, précis, rassurant — never generic, never infantilisant

**Empty state:** `CoachEmptyState` widget already handles the no-profile case with "Pas encore de profil. Trois questions, et on discute." — no new empty state needed for Phase 2 scope.

**Error state:** Per D-08, unknown tool names are silently swallowed in production. No user-visible error for dispatch failures. The `toolDispatchErrorBanner` key above is dev-only insurance.

**Destructive actions in this phase:** None. Phase 2 does not include delete, reset, or irreversible operations.

## Accessibility Contract

- All tappable cards: minimum 44px touch target height (iOS HIG)
- `RouteSuggestionCard` CTA: `Semantics` label derived from route name + "Ouvrir" — not just the button text alone
- `MintScoreGauge` already implements `scoreGaugeSemanticsLabel` ARB key with full score + level + section values
- Color contrast: all text tokens verified WCAG AA in `colors.dart` comments (textMuted fixed to #737378 at 4.6:1, success fixed to #1A8A3A at 4.8:1)
- Widget entrance animation: `ChatCardEntrance` is purely cosmetic — underlying widget is accessible immediately regardless of animation state
