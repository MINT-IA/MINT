static Widget _buildRouteSuggestion(
    BuildContext context, Map<String, dynamic> p) {
  final route = p['route'] as String? ?? '';
  final contextMessage = p['context_message'] as String?
      ?? p['narrative'] as String?
      ?? '';
  final prefill = p['prefill'] as Map<String, dynamic>?;
  if (!ToolCallParser.isValidRoute(route)) return const SizedBox.shrink();
  return RouteSuggestionCard(
    contextMessage: contextMessage,
    route: route,
    prefill: prefill,
    isPartial: p['is_partial'] as bool? ?? false,
  );
}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM output -> ChatToolDispatcher | Untrusted tool call names and arguments from LLM response |
| Tool call arguments -> GoRouter navigation | Route strings from LLM must not reach GoRouter without validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | ChatToolDispatcher.resolveRoute | mitigate | Validate route via ToolCallParser.isValidRoute() whitelist (68 known-good routes). Reject all others with null return. |
| T-02-02 | DoS | ChatToolDispatcher.normalize | mitigate | Cap at _maxToolCallsPerResponse = 5. Prevents LLM flooding the message bubble with excessive widgets. |
| T-02-03 | Tampering | WidgetRenderer._buildRouteSuggestion | mitigate | Check ToolCallParser.isValidRoute(route) before constructing RouteSuggestionCard. Return SizedBox.shrink() for invalid routes. |
| T-02-04 | Information Disclosure | ChatToolDispatcher | accept | Tool call arguments may contain user financial data (amounts, routes). All data stays client-side in Flutter widgets — no network transmission. Low risk. |
</threat_model>
