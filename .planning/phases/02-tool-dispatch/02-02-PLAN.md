<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM output -> ChatMessage.richToolCalls | Tool calls from untrusted LLM now populate the message model directly |
| richToolCalls -> WidgetRenderer | Each RagToolCall is rendered — malformed data must not crash the app |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering | coach_chat_screen.dart SLM path | mitigate | ChatToolDispatcher.normalize() validates and caps tool calls before they enter ChatMessage.richToolCalls. Route validation via ToolCallParser.isValidRoute() in WidgetRenderer (Plan 01). |
| T-02-06 | Tampering | coach_chat_screen.dart BYOK path | mitigate | Same normalization pipeline — ChatToolDispatcher.normalize() applied to parseResult.toolCalls from BYOK response text markers. |
| T-02-07 | DoS | CoachMessageBubble rendering | mitigate | Cap of 5 tool calls per message (from ChatToolDispatcher) prevents rendering loop from creating excessive widgets. |
| T-02-08 | Elevation of Privilege | Keyword matching removal | accept | Removing CoachRichWidgetBuilder eliminates a path where user text controlled which widgets appeared. After removal, only LLM tool decisions (server-influenced) control widgets — reduced attack surface. |
</threat_model>
