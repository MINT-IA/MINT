On the first baseline run (before any Phase 1.5 code change), two gates were already red **on HEAD of `feature/v2.2-p0a-code-unblockers`**, not caused by the rename:

1. **flutter test:** 8991 passed, 2 skipped, **3 failed**.
2. **ruff check (backend):** **6 pre-existing E402 errors** in `services/backend/app/routes/wizard.py`.

The executor escalated to the user before proceeding. The user approved **Option 3** (revised gate protocol, see `01.5-SUMMARY.md` § Scope amendments):

- Flutter test: allowlist semantics. Aggregate passed ≥ 8991, failures ⊆ the 3 baseline failures below.

- Backend commits skip flutter test; end-of-phase full run covers it.
- Targeted test subsets per commit for F-layers.

## Julien/Lauren `** FAIL **` markers — investigation result

The initial baseline log showed output like:
```
** FAIL ** | LPP Julien rente
       Actual:   45954.01
       Expected: 33892.00
** FAIL ** | LPP Lauren balance
       Actual:   203570.02
       Expected: 153000.00
** FAIL ** | Taux remplacement
       Actual:   44.75
       Expected: 65.50
```

User asked whether these are real regressions (would warrant a P1 STAB item) or noisy diagnostic logs from a passing harness.

**Investigation (≤10 min, on commit `a6a2b99b` HEAD):**
- File: `apps/mobile/test/golden/golden_couple_validation_test.dart`
- Line 43-55: helper `_verdict(label, actual, expected, {tolerancePct})` returns a **string** of the form `"$status | $label\n  Actual: ...\n  Expected: ...\n  Delta: ..."`. The helper does NOT call `expect()` internally. The "** FAIL **" prefix is purely a printf status marker.
- Line 255-267: the LPP Julien test calls `print(_verdict(...))` to emit the diagnostic, then follows with loose assertions `expect(projectedBalance, greaterThan(200000))` and `expect(annualRente, greaterThan(10000))`. Both pass.
- Line 299-303: LPP Lauren test prints the verdict, then asserts `expect(projectedBalance, greaterThan(50000))`. Passes.
- Line 542-550: Taux remplacement test prints verdict, then asserts `> 30%` and `< 120%`. Both pass (44.75 is within bounds).

**Conclusion: the `** FAIL **` markers are diagnostic-only. No real assertion failure. The Julien/Lauren golden couple calculations are drifting from the documented CLAUDE.md golden values (e.g., LPP Julien rente 33 892 expected vs. 45 954 actual), but the tests themselves are passing because the harness tolerances are generous (25% for LPP, 30% for Taux remplacement).**

**NO new STAB item filed.** However, this drift is a real signal that the Julien/Lauren calculations have moved away from the CLAUDE.md §8 documented expected values. It is **recommended** (not required) that a separate sprint:
1. Decide whether the CLAUDE.md expected values need updating to match current calculations, OR the calculations need restoring to match the CLAUDE.md expected values.
2. Replace the `print(_verdict(...))` + loose-sanity pattern with real hard assertions at tight tolerance (5%), so future drift becomes immediately visible.
3. File as `STAB-CARRYOVER: Julien/Lauren golden couple drift signal (diagnostic-only, non-blocking)` in `.planning/backlog/STAB-carryover.md` when that backlog file is created.
