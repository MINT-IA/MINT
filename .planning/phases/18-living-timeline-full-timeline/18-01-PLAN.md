**2. Create `apps/mobile/lib/providers/timeline_provider.dart`:**

`TimelineProvider extends TensionCardProvider` (per CONTEXT.md decision). Override `refresh()`:
- Call `super.refresh()` first (populates tension cards + loop position)
- Then aggregate nodes from the SAME data already fetched by parent. To avoid double-fetching, refactor: extract the raw data fetch into a protected method `fetchRawData()` that returns a record `({commitments, landmarks, conversationEntries, partner})`. Parent's `refresh()` calls it, child's `refresh()` calls `super.refresh()` then uses cached raw data via a protected getter `_lastRawData`.

Node aggregation logic:
- **CommitmentNodes**: Each commitment from `getCommitments()` becomes a node. Status `completed` -> `TensionType.earned`, status `active` -> `TensionType.pulsing`. Deep link: `/coach/chat?prompt=Montre-moi+mes+engagements`. Title i18n key: `timelineCommitmentEarned` or `timelineCommitmentActive`. Subtitle: the `whenText` or `ifThenText`.
- **ConversationNodes**: Each entry from SharedPreferences conversation index becomes a node. All past conversations -> `TensionType.earned`. Deep link: `/coach/chat`. Title i18n key: `timelineConversation`. Subtitle: the conversation title.
- **CoupleNode**: If `partner != null && partner.isDeclared`, create ONE node with `TensionType.earned`. Deep link: `/coach/chat?prompt=Questions+pour+mon+partenaire`. Title i18n key: `timelineCoupleEstimate`. Subtitle: empty.
- **ProjectionNodes** (future/ghosted): Create 1-2 from landmarks. Each landmark -> `TensionType.ghosted`. Deep link: `/coach/chat?prompt=Parle-moi+de+mon+prochain+jalon`. Title i18n key: `timelineProjection`. Subtitle: `landmark.message`. Date: parsed from `landmark.date`.
- **DocumentNodes**: Read from SharedPreferences key `_uploaded_documents` (list of `{id, name, uploadedAt}`). If key doesn't exist, skip (empty list). Each -> `TensionType.earned`. Deep link: `/explorer`. Title i18n key: `timelineDocument`. Subtitle: document name.

After collecting all nodes:
- Sort by `date` descending (newest first)
- Cap at 50 nodes (pagination: expose `bool hasMore` and `loadMore()` method that increases cap by 20)
- Group into `List<TimelineMonth>` by year+month. Mark current month's group as `isCurrentMonth = true`.

Expose:
- `List<TimelineMonth> months` — grouped, sorted
- `bool hasMore` — more nodes available
- `void loadMore()` — increases visible cap, re-groups, notifyListeners
- `bool get hasNodes` — at least 1 node exists

      SliverToBoxAdapter(child: MonthHeaderWidget(month: month, ...)),
      if (!isCollapsed(month))

    // ── Empty timeline state ──
    if (!provider.hasNodes)
      SliverToBoxAdapter(child: _EmptyTimeline()),

    // ── Bottom padding ──
    SliverToBoxAdapter(child: SizedBox(height: 80)),
  ],
)
