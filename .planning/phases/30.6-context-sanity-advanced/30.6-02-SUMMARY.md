### Auto-fixed Issues

**1. [Rule 1 - Bug] `sentry_flutter 9.14.0` API surface corrected**
- **Found during:** Task 2 (spike execution)
- **Issue:** `.planning/research/STACK.md §A.1` and 30.6-02 plan described the Replay + privacy API as `options.experimental.replay.{maskAllText,maskAllImages,sessionSampleRate,onErrorSampleRate}` + `options.tracePropagationTargets = [...]`. On first `flutter analyze` after editing `main.dart`, the compiler flagged 5 errors:
  - `undefined_getter` on `options.experimental` (×4)
  - `assignment_to_final` on `tracePropagationTargets`
- **Root cause:** the installed library exposes:
  - privacy masks on `options.privacy.*` (class `SentryPrivacyOptions`), NOT on `.experimental.replay`
  - replay sampling rates on `options.replay.*` (class `SentryReplayOptions`), also NOT under `.experimental`
  - `tracePropagationTargets` is declared `final List<String>` with default `['.*']` → must be mutated in place (`..clear()..addAll(...)`), not reassigned
- **Fix:** Rewrote the `SentryFlutter.init` options closure to use the actual 9.14.0 surface. Verified via `grep` of `~/.pub-cache/hosted/pub.dev/sentry_flutter-9.14.0/lib/src/sentry_{flutter_options,privacy_options,replay_options}.dart` before the second edit. Pattern intent preserved: masks ON, replay sampling low, trace propagation narrowed.
- **Files modified:** `apps/mobile/lib/main.dart` (only; same atomic commit)
- **Verification:** `dart format --set-exit-if-changed` exit 0; `flutter analyze lib/main.dart` → `No issues found! (ran in 4.0s)`. 4 mandatory greps (`sentry_flutter: 9.14.0`, `maskAllText`, `maskAllImages`, `SentryWidget`) all match.
- **Committed in:** `38a3950b` (the only spike commit; first correct try inside Task 2, 1 auto-fix under the 3-attempt cap)
- **Why flagged Rule 1 not Rule 4:** API shape is a library implementation detail, D-18 intent (masks ON, replay sampling low, allowlisted propagation) fully preserved. No architectural decision changed.
