# Phase 26 Plan 01: Navigation Coherence — Final Fixes

## Objective

Fix all remaining navigation issues from Gate 0 walkthrough: auth-state propagation to tabs, lightning menu route handling, and back navigation consistency.

## Context

Gate 0 findings (from `.planning/architecture/14-INFRA-AUDIT-FINDINGS.md`):
- P0-1: Auth state not propagating — Explorer+Aujourd'hui show wrong content
- P0-3: Lightning menu "Scanner un document" does nothing
- P0-4: Back navigation trapping

Most P0s were fixed in phases 9-25. This phase verifies remaining issues and fixes what's left.

## Tasks

<task id="1" type="auto">
<name>Fix lightning menu route handling — scan and profile routes</name>
<description>
The lightning menu's `onNavigate` callback in coach_chat_screen.dart uses `ChatDrawerHost.resolveDrawerWidget(route)` which returns null for `/scan` and `/profile/bilan`. When null, nothing happens — the menu closes and the user sees no response.

Fix: For routes that should NOT be drawers (like `/scan` which needs camera access, or `/profile/bilan`), fall back to `context.push(route)` instead of silently swallowing the navigation.

Also fix `_lazyScreen` in chat_drawer_host.dart — it returns empty Containers instead of actual widgets. Either wire real screen imports or remove the drawer approach for these routes and use push navigation.
</description>
<done>
- Lightning menu "Scanner un document" navigates to document scan screen
- Lightning menu "Completer mon profil" navigates to financial summary screen
- Routes that have no drawer widget fall back to push navigation
</done>
</task>
