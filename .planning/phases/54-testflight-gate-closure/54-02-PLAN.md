# Plan 54-02 — `intentTag` → tappable chip on every coach opener (PLAN)

**Status:** Ready (must ship BEFORE Plan 54-01 walker run per CONTEXT D-02)
**Companion CONTEXT:** `54-CONTEXT.md`
**Estimated effort:** 2 days

## Goal (one sentence)

Every coach opener (proactive trigger, sequence-step prompt, precomputed insight) that today emits a plain text bubble must offer the user a tap-to-act chip when an `intentTag` is available — closing the « 3 already-working coach intelligences with 0 user-visible chip » gap caught by Expert 4 in the post-Phase-53 panel.

## Why this plan ships before Plan 54-01 walker

Plan 54-01's autonomous walker drives sim taps across `AUDIT_TAP_RENDER.md`. For coach-opener rows (proactive trigger banners, silent opener cards, sequence-step prompts), there's currently nothing to tap because the chip wiring is missing. Walker would PASS those rows trivially (« text bubble visible ») while masking the actual user-experience gap. Plan 54-02 makes the gap visible to the walker.

## Current state (read 2026-05-04)

Per Expert 4's evidence (`.planning/decisions/2026-05-04-phase-54-target.md` Coach Intelligence Architect verdict):

1. `apps/mobile/lib/screens/coach/coach_chat_screen.dart:502-514` — `_addCoachOpenerMessage(String content)` writes a plain `ChatMessage` with no `intentTag`, no `responseCards`, no `routeSuggestion`. **The intentTag is dropped on the floor.**
2. `apps/mobile/lib/screens/coach/coach_chat_screen.dart:521` — `_resolveProactiveOpener` returns ONLY a `String`, even though `ProactiveTrigger.intentTag` is populated by 7/8 trigger types in `proactive_trigger_service.dart:75-98`.
3. `_injectSequencePrompt` (Plan 53-02) DOES emit a `route_to_screen` rich tool call alongside the text — that path works. The remaining gap is the proactive trigger + precomputed insight branches.
4. `apps/mobile/lib/services/coach/precomputed_insights_service.dart:275-292` — `getCachedInsight(prefs:)` exists with `intentTag` field populated, but **0 UI consumers** in `lib/screens/`.

### T-54-02-01 — Refactor `_addCoachOpenerMessage` to accept optional intentTag

**Files:** `apps/mobile/lib/screens/coach/coach_chat_screen.dart`

**Test:** unit test in `apps/mobile/test/screens/coach/coach_chat_screen_opener_test.dart` (new) — assert chip renders when intentTag passed; assert plain bubble when not.

## Risks + mitigations

| Risk | Mitigation |
|---|---|
| `_addCoachOpenerMessage` callers (proactive + sequence + precomputed insight) end up shipping THREE chips on a chat-open burst → spam | Apply « 1 opener chip per chat-open » rule: precomputed insight wins when present; otherwise proactive trigger; otherwise none. The sequence-step chip remains independent (it fires post-screen-return, not on chat-open). |
| Precomputed insight has stale data after a profile update | `getCachedInsight` already has staleness logic; trust it. If staleness is broken, file as a Phase 55+ follow-up — not a 54-02 blocker. |
| New ARB keys cause parity drift across 6 locales | Pre-push checklist enforces `flutter gen-l10n` + key-count assertion (per `feedback_pre_push_checklist.md`). Verify ARB key counts match before push. |
| `RouteSuggestionCard` re-render on chip tap consumes the underlying tool call → tap fires `SequenceChatHandler.startSequence` for an opener that wasn't supposed to start a sequence (e.g. precomputed insight not keyed to a SequenceTemplate) | This is the no-op path: `SequenceChatHandler.startSequence` returns null for unknown intents, no SequenceRun persisted (verified by `route_suggestion_card_sequence_test.dart` test 3). Safe by construction. |
