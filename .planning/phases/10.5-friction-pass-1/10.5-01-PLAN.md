Create `docs/FRICTION_PASS_1/README.md` — a single-page, print-friendly, phone-readable instruction card for Julien. This is read ONCE by Julien on his laptop before touching the device, then followed.

Structure (use headings and short bullets, no prose walls):

## Before you start (5 min, on laptop)
1. Confirm branch: `git branch --show-current` → `feature/v2.2-p0a-code-unblockers`
2. Plug Galaxy A14 via USB. Enable USB debugging (Developer Options → USB debugging).
3. `adb devices` → should show one device, state `device`.
4. Run `tools/scripts/build_a14.sh --install --launch`
5. Verify app icon appears in app drawer. Force-stop the app (Settings → Apps → MINT → Force stop). Do NOT keep it warm.

## Recording setup (2 min, on device)
1. Swipe down twice from top → find "Screen record" tile (add to quick settings if missing).
2. Enable "Show taps" in Developer Options → "Input" section. This draws tap circles in the recording.
3. Enable "Pointer location" OFF (too cluttered).
4. Mute notifications (Do Not Disturb on).
5. Open a stopwatch on a SECOND device (Mac iPhone app, wristwatch, or second phone). You need to time cold-start without touching the A14.

## The walkthrough (30 min, on device) — 3 runs
For EACH run (do this 3 times, cold-start each time):

1. Force-stop MINT in Android settings. Wait 10 seconds.
2. On the second device, prepare stopwatch at 00:00.
3. Start screen recorder on A14.
4. Move finger over the MINT icon but DO NOT tap yet.
5. Start stopwatch and tap icon simultaneously.
6. Observe S0 landing. Note the stopwatch value when you see first pixel of S0 → this is `cold_start_ms` (target <2500ms).
7. Note when intent chips become tappable (tap one and see state change) → this is `interactive_ms` (target <3000ms).
8. Tap the "explore" intent chip.
9. On the chat screen, type EXACTLY: `Je viens d'avoir 30 ans, je commence à me demander si je devrais ouvrir un 3a.`
10. Tap send and reset stopwatch simultaneously.
11. Note when the first character of the assistant response appears → this is `first_reply_ms` (target <4000ms).
12. Read the full response once.
13. Stop screen recording.
14. Open `docs/FRICTION_PASS_1.md` on your phone (or a paper notebook) and log every frottement you noticed during this run — use the table template, one row per frottement.

## After the 3 runs
1. `adb pull /sdcard/Movies/<each recording file> docs/FRICTION_PASS_1/walkthrough_run{1,2,3}.mp4`
2. If files >100MB: `ffmpeg -i input.mp4 -vf fps=2 docs/FRICTION_PASS_1/frames_run1/frame_%04d.png` (Claude will do this if you can't)
3. Finalize `docs/FRICTION_PASS_1.md` — the three cold-start/interactive/first-reply numbers go in a "Perf numbers" section at the top.
4. Tell Claude: "Walkthrough done, triage please." → Plan 10.5-03 begins.

## The shame test (MANDATORY mental check per frottement)
Per CONTEXT.md D-04: ask yourself for every note you consider — *"If a 55-year-old Swiss grandmother with no finance background saw this, would she think 'I'm stupid' or 'I'm behind'?"* If yes → **block**. Not polish. Not nit. **Block.**

## Severity (LITERAL, from D-04)
- **block** = user can't proceed OR feels shame OR perf metric fails OR banned term visible OR >5s unresponsive
- **polish** = noticeable, doesn't break flow, no shame leak
- **nit** = only Claude would notice

## Golden path is CLOSED scope
Per D-09: you are testing exactly: cold start → S0 → intent chip → chat opener → first message → first insight. You are NOT exploring ProfileDrawer, Explorer, life-event screens, etc. Stay on the golden path.
