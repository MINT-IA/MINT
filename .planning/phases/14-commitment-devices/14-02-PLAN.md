<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Flutter → backend API | Commitment creation request crosses from client to server |
| Backend API → DB | User-submitted text stored in database |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-06 | Elevation of Privilege | IDOR on PATCH /commitment/{id} | mitigate | Query filters by both commitment_id AND user_id from JWT — user cannot modify other users' commitments |
| T-14-07 | Tampering | Malicious text in commitment fields | mitigate | Server-side length validation (500 char max per field) + PII patterns scrubbed before storage |
| T-14-08 | Denial of Service | Unbounded commitment creation | mitigate | Rate limit: max 50 pending commitments per user (enforce in POST endpoint) |
| T-14-09 | Information Disclosure | Commitment data in notification payload | accept | Deeplink payload contains only commitment ID, not text content — actual data fetched from server on tap |
</threat_model>
