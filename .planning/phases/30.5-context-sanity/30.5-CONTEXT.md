### CLAUDE.md restructure topology (CTX-03)

  1. **Banned terms** (compliance LSFin)
  2. **Accents 100% mandatory** (`creer→créer`, `eclairage→éclairage`, etc.)
  3. **Retirement framing prohibition** (identity-critical, MINT ≠ retirement app)
  4. **Financial_core reuse mandatory** (`lib/services/financial_core/`, jamais ré-inventer)
  5. **i18n required** (toutes strings user-facing via `AppLocalizations`)
- **D-07:** 10 NEVERs à convertir en triplets `{bad → good → why}` (Min 2022, +15-25pts recall évité), ordonnés par fréquence empirique de violation (extraite des feedback_*.md MEMORY.md) :
  1. Hardcode strings (i18n drift) — le plus fréquent
  2. Hardcode colors (`MintColors.*` skip)
  3. Duplicate calculation logic (`financial_core/` ré-invention)
  4. Frame MINT as retirement app
  5. Use banned terms
  6. Code without reading existing code (façade-sans-câblage doctrine #1)
  7. Assume Swiss native (archetype skip)
  8. Promise returns (no-promise compliance)
  9. Projection without confidence score
  10. Skip tests
- **D-08:** Audit redondance scope = **CLAUDE.md §5-7 vs skills `mint-swiss-compliance` + `mint-flutter-dev` + `mint-backend-dev` uniquement** (Panel B finding : double-injection mesurable). Critère redondance = "le contenu apparaît mot-à-mot OU sémantiquement équivalent dans les 2 surfaces". Ré-architecture complète skills ↔ CLAUDE.md = **deferred v2.9+** (out of scope CTX).

### Drift dashboard (CTX-02)
- **D-09:** Surface = **CLI Python + nightly markdown report**, PAS mobile route `/admin/agent-drift`. REQ baseline avait un slip vers le pattern Phase 32 `/admin/routes` (mimétisme), override panel décidé : c'est un meta-tool dev-only (les users mobile s'en foutent), Flutter screen serait gaspillage. Implémentation : `tools/agent-drift/dashboard.py` (CLI invocable `python3 tools/agent-drift/dashboard.py [--baseline|--report|--ingest]`) → génère `.planning/agent-drift/YYYY-MM-DD.md` automatiquement chaque nuit (cron local ou lefthook post-commit batch).
- **D-10:** Storage = **SQLite local `.planning/agent-drift/drift.db`** (write-heavy short-term, read-low). Postgres backend overkill pour dev tool, JSON files inadaptés au query (4 métriques agrégées). SQLite single-file, gitignored, rebuilt-friendly.
- **D-11:** 4 métriques data sources :
  - **(a) drift rate** = parse `git log --author='Co-Authored-By: Claude'` last 7d × grep `tools/checks/{accent_lint_fr,no_hardcoded_fr,no_bare_catch}.py` post-hoc sur les commits → `% commits agent avec ≥1 violation`. Implémentable J1-J2 sans dépendance lints Phase 34 (early-version of `accent_lint_fr.py` ship en CTX-02 si nécessaire).
  - **(b) context hit rate** = log local quand `gsd-prompt-guard.js` détecte une violation rule pré-1er-tool_use. Extension du hook existant, pas Sentry breadcrumb (Sentry breadcrumb suppose Phase 31 déjà shipped). Stocké `.planning/agent-drift/context_hits.jsonl`.
  - **(c) token cost per session** = parse `.claude/projects/-Users-julienbattaglia-Desktop-MINT/*.jsonl` transcripts (déjà sur disque, 0 API call Anthropic) → somme `usage.input_tokens + usage.output_tokens` par session.
  - **(d) time-to-first-correct-output** = harness offline 20 golden prompts (run sur `claude --headless` ou équivalent) → mesure # tours avant 1er output validant accents/banned terms/i18n via les mêmes lints que (a). Run nightly ou on-demand.
- **D-12:** Baseline J0 = **single snapshot pré-refonte** (avant CTX-03 commits) capturé `.planning/agent-drift/baseline-J0.md`. Rolling 7j = post-baseline tracking. Évite le moving-target effect.

### UserPromptSubmit hook (CTX-04)
- **D-13:** Hook script = **Node.js** (convention `.claude/hooks/*.js` déjà établie : `gsd-prompt-guard.js`, `context-monitor.js`, etc.). Nouveau fichier `.claude/hooks/mint-context-injector.js`. Pas Python (différent runtime), pas Bash (limites parsing JSON stdin/stdout des hooks).
- **D-14:** 5 patterns regex (REQ baseline, ordre = priorité injection top-3) :
  1. `\.arb` édité dans le user message (file path mention) → inject ARB parity reminder
  2. `\.dart` dans `screens/` mentionné → inject i18n + accent + MintColors reminder
  3. `\b(calcul|calculator|simulator|projection|rente|capital|tax)\b` (case-insensitive) → inject `financial_core` reuse reminder
  4. `\bcommit\b` (verbe) → inject commit hygiene reminder (no-bare-catch, branch flow, conventional commits)
  5. `\bnouveau\s+\.dart|create.*\.dart\b` → inject existing-code-check reminder (façade-sans-câblage doctrine, "lis avant d'écrire")
- **D-15:** Dédup multi-match = **top 3 par priorité ordre source code hook** (D-14 ordre = priorité). JAMAIS plus de 3 simultanés. Header dédup unique "📌 MINT context — N rules apply" puis snippets concat. Total <800 tokens worst case (3×~250 tokens). Évite pollution context Panel A <2k threshold.
- **D-16:** Snippets location = **fichiers séparés `.claude/context-snippets/{arb,screens,calculator,commit,new-dart}.md`** (5 fichiers, ~200-400 tokens chacun). Hook lit lazy au match. Modulaire, éditable sans toucher `.js`. Permet itération copy sans déployer code.
- **D-17:** Override mécanisme = **env var `MINT_NO_CONTEXT_INJECT=1`** (grep-able shell history per doctrine GUARD-07 `LEFTHOOK_BYPASS=1` pattern). Pas de flag CLI (Claude Code ne passe pas args aux hooks). Hook check `process.env.MINT_NO_CONTEXT_INJECT === '1'` → no-op.
