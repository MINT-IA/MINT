## Summary

Phase 30.5 est une **refonte éditoriale + instrumentation runtime**, pas une feature produit. Les 5 livrables s'articulent autour d'un contrat simple :

1. Les agents doivent pouvoir **lire** le contexte qui leur est destiné (`MEMORY.md` tronqué silencieusement à ~200L runtime = bug runtime P0 confirmé Panel A — `CTX-01`).
2. Le **drift doit être mesurable** avant toute itération prompt-engineering (Panel C : "sans métriques, on pilote à l'instinct" — `CTX-02`).
3. Le **lost-in-the-middle** (Liu 2024) se mitige par bracketing TOP+BOTTOM + remplacement des `NEVER` par triplets démonstratifs (Min 2022, +15-25pts recall — `CTX-03`).
4. L'**attention conditionnelle** se gère côté runtime via `UserPromptSubmit` hook qui injecte 200-400 tokens ciblés selon patterns MINT détectés (`CTX-04`).
5. Le **spike fresh-context** sur une tâche réaliste Phase 31 (sentry 8→9, maskAllText) valide go/no-go ; sinon fallback Modeste 1 (`CTX-05`).

**Primary recommendation:** Implémenter dans l'ordre **CTX-02 (dashboard J0 baseline) → CTX-01 (MEMORY split + lefthook skeleton) → CTX-03 (CLAUDE.md restructure) → CTX-04 (hook) → CTX-05 (spike)**. CTX-02 **AVANT** CTX-01/CTX-03 : le dashboard doit capturer la baseline J0 AVANT toute refonte (D-12). Tout inverser = baseline polluée = moving target.

**Confidence Panel D's concern (perfection paralysie):** adressée par kill-policy D-01 (fallback Modeste 1 si spike fail 2×) + budget 5j non-empruntable (pas 14j).

### Pattern 1: MEMORY INDEX + topics lazy-load (CTX-01)

**What:** Le fichier `MEMORY.md` est au home (pas dans le repo MINT). Il est chargé automatiquement au `SessionStart` par le Claude Agent SDK. Runtime tronque silencieusement >~200L (cause racine confirmée Panel A).

**Implementation:**
- `MEMORY.md` devient INDEX-only (~100L cible D-02, HARD gate en warning, vrai critère = 30j retention).
- Détails dans `memory/topics/{type}_{slug}.md` — naming convention DÉJÀ en place sur 60+ fichiers, migration = `mv memory/feedback_*.md memory/topics/` (aucun refactor de contenu).
- Les `[feedback_xxx](feedback_xxx.md)` links dans l'INDEX doivent être updated en `[feedback_xxx](topics/feedback_xxx.md)` sinon liens cassent.
- Les `feedback_*.md` ne sont **jamais chargés automatiquement** (Panel A). L'INDEX fait office de table des matières ; l'agent doit `Read` explicitement.

**Migration path:**
```bash
cd ~/.claude/projects/-Users-julienbattaglia-Desktop-MINT/memory/
mkdir -p topics archive
# 1. Move all feedback/project/reference/user files (60+) to topics/
for f in feedback_*.md project_*.md reference_*.md user_*.md; do
  mv "$f" topics/
done
# 2. Rewrite MEMORY.md links: (xxx.md) → (topics/xxx.md)
# Handled via tools/memory/migrate_links.py (ship dans CTX-01)
# 3. Verify no broken links: tools/memory/verify_links.py
```

### Pattern 4: UserPromptSubmit hook contract (CTX-04)

**What:** Claude Agent SDK hook event (Panel A §1) qui reçoit le prompt user via stdin JSON, peut **prepend** du texte au prompt avant que le modèle le voie. C'est le seul vrai moyen d'injecter du contexte mid-session côté runtime.

**When:** Sur 5 patterns regex MINT (D-14). Hook est sync (timeout <3s), ne doit jamais bloquer.

**Stdin JSON schema** (vérifié via `.claude/hooks/gsd-prompt-guard.js` pattern) :

```json
  "tool_name": "…",           // empty for UserPromptSubmit

}
```

**Stdout JSON schema** for prepend:

### Anti-Patterns to Avoid

- **AST proof-of-read PreToolUse** — théâtre (Panel A §4 + D §4). Permanent KILL, pas deferred.
- **`.cursorrules` monolithique racine** — déprécié Cursor 2025 (Panel B §1). Permanent KILL.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Pre-commit parallel runner | Custom git hooks bash | **lefthook 2.1.5** (brew) | Single Go binary, parallel by default, polyglot-native, monorepo-native (STACK.md §C.1). JAX / Evil Martians precedent. |
| SQL storage | Hand-rolled JSON+grep | **SQLite stdlib Python** | `sqlite3` en stdlib Python 3, 0 install. Query agrégé 4 métriques trivial. |
| Hook input/output parsing | Custom bash JSON | **Node.js JSON.parse** | Convention `.claude/hooks/*.js` établie. Pattern validé `gsd-prompt-guard.js`. |
| MEMORY.md runtime truncation detection | Read + grep after each session | **lefthook pre-commit check** `tools/checks/memory_retention.py` | Mécanique, pas prose. Block le commit si >30j entries non-archivées. |
| Git log parsing for agent commits | Custom regex bash | **`git log --format='%H %ae %s' --author='Co-Authored-By: Claude'`** | Git natif, stable, testable. |
| Claude Code JSONL transcript parsing | Regex bash | **Python `json.loads()` per line** | JSONL format = 1 JSON obj/line, `.message.usage.{input_tokens, cache_creation_input_tokens, cache_read_input_tokens, output_tokens}` schema `[VERIFIED: live parse 2026-04-19]` |
| CLI tool generation | Argparse from scratch | **Python `argparse` + sub-commands** | Stdlib, 0 dep, `--baseline/--report/--ingest` mutually exclusive. |
| Fresh-context spike launch | "Remember to forget" in same session | **New terminal + `claude` cold start on branch `feature/v2.8-ctx-spike-30.5`** (D-19) | Session state is process-local. Cold start = guaranteed 0 carryover. |

**Key insight:** Chaque composant Phase 30.5 a un équivalent stdlib ou binaire éprouvé. Le risque n'est pas la technologie, c'est la **discipline de câblage** (façade-sans-câblage, doctrine #1 MEMORY.md). Le lefthook skeleton (D-04) doit **prouver qu'il RUNS** sans activer de lint — c'est le test critique de "rails posés mais non sous tension".

## Common Pitfalls

### Pitfall 1 — Façade-sans-câblage (doctrine #1 MINT)
**What goes wrong:** Skeleton `lefthook.yml` écrit mais jamais vérifié qu'il exécute. Un commit passe vert alors que le hook silencieusement ne tourne pas.
**Why it happens:** `lefthook install` pas appelé post-install ; ou `lefthook.yml` parse mais commands array vide.
**How to avoid:**
- Test dry-run obligatoire : `lefthook run pre-commit --files <stagedfile>` doit exit non-zero si la règle MEMORY gate échoue sur un test fixture.
- Ship `tools/checks/lefthook_self_test.sh` qui crée un fichier MEMORY temp avec 31j old entries, stage, commit attempt → doit FAIL.
**Warning signs:** lefthook.yml présent mais `.git/hooks/pre-commit` absent ou vide → `lefthook install` jamais exécuté.

### Pitfall 2 — Baseline J0 polluée (mesure CTX-02 inutile)
**What goes wrong:** Dashboard CTX-02 développé EN MÊME TEMPS que restructure CTX-03 → baseline J0 capture déjà l'effet du refactor → dashboard "montre" -30% drift alors que c'est l'observateur qui a changé la mesure.
**Why it happens:** Build order non-respecté. Le CONTEXT.md (D-12) est explicite : "baseline J0 = single snapshot pré-refonte".
**How to avoid:**
- **Hard sequencing CTX-02 avant CTX-03** dans le plan. Pas de PR CTX-03 tant que `baseline-J0.md` pas committé.
- Dashboard ingest uniquement commits avec date ≥ baseline capture date.
**Warning signs:** Variance des 4 métriques > 30% entre jours consécutifs post-baseline sans aucune refonte shipped → baseline probablement capturée après partial refonte.

### Pitfall 3 — Hook UserPromptSubmit silent fail
**What goes wrong:** Le hook crash (malformed JSON, missing snippet file, regex error) mais stdin/stdout pipe pattern fait que Claude Code continue sans le context injecté → drift silencieux.
**Why it happens:** Convention hooks `.claude/hooks/*.js` utilise `process.exit(0)` dans catch — never-block philosophy (vérifié `gsd-prompt-guard.js` L92-95 `catch { process.exit(0); }`).
**How to avoid:**
- **Log-on-failure** : si catch déclenché, écrire une ligne `.planning/agent-drift/hook_failures.jsonl`. Silent fail acceptable RUNTIME, mais DOIT laisser trace disque pour debug.
- Unit tests Node.js simulent 10 cas : JSON malformed, fichier snippet manquant, regex qui throw, env var override set, no match, 1 match, 2 matches, 3 matches, 4 matches (→ top-3 dédup), 5 matches (→ top-3).
**Warning signs:** `hook_failures.jsonl` grossit > 1 ligne/session → bug à fixer.

### Pitfall 4 — Double-injection CLAUDE.md §5-7 + skill (Panel B)
**What goes wrong:** Post-restructure CTX-03, CLAUDE.md §5 business rules garde les constantes 2025/2026 + `mint-swiss-compliance` skill garde les banned terms → agent charge 2× la même info, context pollution (~800 tokens gaspillés).
**Why it happens:** Redondance non-auditée. D-08 scope = limité à CLAUDE.md §5-7 vs skills `mint-*`.
**How to avoid:**
- Script `tools/audit/claude_md_vs_skills.py` (ship dans CTX-03) : extrait sections §5 (BUSINESS RULES), §6 (COMPLIANCE), §7 (UX RULES) et pour chaque ligne non-trivial grep dans `.claude/skills/mint-*/SKILL.md` (fuzzy match sémantique = 80% identité mot-à-mot).
- Ouvrir markdown report listing les duplicates ; pour chaque, décision : garder CLAUDE.md 1-ligne résumé, déplacer détail → skill.
**Warning signs:** CLAUDE.md post-refonte >150L alors que même règles sont dans `mint-swiss-compliance/SKILL.md` → pas d'audit fait.

### Pitfall 5 — Spike CTX-05 biais d'amorçage
**What goes wrong:** L'agent qui fait le spike a travaillé sur la restructure CLAUDE.md juste avant → se "souvient" des règles même si la nouvelle CLAUDE.md est cassée → spike passe, faux positif.
**Why it happens:** Mémoire process-local d'une session Claude Code contient tous les tool_use et contenus lus.
**How to avoid:** D-19 — new terminal, `claude` cold start, branche dédiée `feature/v2.8-ctx-spike-30.5` checked out depuis dev frais. **Warning signs:** L'agent mentionne spontanément des détails de la restructure CLAUDE.md qu'il ne pourrait pas connaître sans l'avoir lue in-session.

### Pitfall 6 — PII dans le spike (A1 research/PITFALLS.md)
**What goes wrong:** Le spike wire `SentryWidget` SANS `maskAllText: true` + `maskAllImages: true` → Session Replay capture revenus/IBAN utilisateurs simulator = violation nLPD + LSFin art. 7-10.
**Why it happens:** `SentryFlutter.init` options.experimental.replay defaults ne sont PAS masked à true.
**How to avoid:** D-18 rend explicite que le spike DOIT inclure `options.experimental.replay.maskAllText = true` + `maskAllImages = true`. Grille 5-dim #1 (CTX-05 pass criteria) vérifie "0 violation compliance" — planner peut étendre grille avec "replay masks set" checkbox.
**Warning signs:** Diff du spike touche main.dart sans mentionner `replay.mask*`.

### Pitfall 7 — 30j retention détecté par frontmatter (pas mtime)
**What goes wrong:** CTX-01 GC tente de lire frontmatter YAML `date:` dans `memory/topics/*.md` → 60+ fichiers pas migrés au format frontmatter, GC dead-stops ou efface par mistake.

**Warning signs:** gc.py importe `yaml` ou `frontmatter` libraries → mauvaise direction, revert vers mtime.

### Pitfall 8 — `flutter clean` ou delete Podfile.lock pendant spike CTX-05
**What goes wrong:** Spike bump `sentry_flutter` sur iOS → macOS Tahoe → `flutter clean` casse Pods resolution → heures perdues à déboguer build iOS.

### Example 1 — UserPromptSubmit hook skeleton (CTX-04)

```javascript
#!/usr/bin/env node
// .claude/hooks/mint-context-injector.js
// UserPromptSubmit hook — injects MINT-specific context snippets
// Source pattern: .claude/hooks/gsd-prompt-guard.js (stdin/stdout JSON)

const fs = require('fs');
const path = require('path');

// D-17: override via env var (grep-able in shell history)
if (process.env.MINT_NO_CONTEXT_INJECT === '1') {
  process.exit(0);
}

// D-14: 5 patterns (ordre = priorité injection)
const PATTERNS = [
  { id: 'arb',        snippet: 'arb.md',        regex: /\.arb\b/i },
  { id: 'screens',    snippet: 'screens.md',    regex: /\.dart\b[^\n]*screens\//i },
  { id: 'calculator', snippet: 'calculator.md', regex: /\b(calcul|calculator|simulator|projection|rente|capital|tax)\b/i },
  { id: 'commit',     snippet: 'commit.md',     regex: /\bcommit\b/i },
  { id: 'new-dart',   snippet: 'new-dart.md',   regex: /\b(nouveau\s+\.dart|create.*\.dart)\b/i },
];

const SNIPPETS_DIR = path.join(__dirname, '..', 'context-snippets');
const LOG_PATH = path.join(__dirname, '..', '..', '.planning', 'agent-drift', 'hook_failures.jsonl');

function logFailure(reason, detail) {
  try {
    fs.mkdirSync(path.dirname(LOG_PATH), { recursive: true });
    fs.appendFileSync(LOG_PATH, JSON.stringify({ ts: Date.now(), reason, detail }) + '\n');
  } catch { /* silent */ }
}

let input = '';
const timeout = setTimeout(() => process.exit(0), 3000);
process.stdin.setEncoding('utf8');
process.stdin.on('data', c => input += c);
process.stdin.on('end', () => {
  clearTimeout(timeout);
  try {
    const data = JSON.parse(input);
    const userPrompt = data.user_prompt || data.prompt || '';
    if (!userPrompt) { process.exit(0); }

    // Match patterns in priority order
    const matches = PATTERNS.filter(p => p.regex.test(userPrompt));
    if (matches.length === 0) { process.exit(0); }

    // D-15: top-3 dedup
    const top3 = matches.slice(0, 3);
    const sections = [];
    for (const m of top3) {
      try {
        const snippet = fs.readFileSync(path.join(SNIPPETS_DIR, m.snippet), 'utf8');
        sections.push(snippet.trim());
      } catch (e) {
        logFailure('snippet_missing', { id: m.id, err: e.message });
      }
    }
    if (sections.length === 0) { process.exit(0); }

    const header = `📌 MINT context — ${top3.length} rule${top3.length>1?'s':''} apply`;
    const body = [header, '', ...sections].join('\n\n');

    process.stdout.write(JSON.stringify({
      hookSpecificOutput: {
        hookEventName: 'UserPromptSubmit',
        additionalContext: body,
      },
    }));
  } catch (e) {
    logFailure('parse_or_logic', { err: e.message });
    process.exit(0);  // never-block philosophy
  }
});
```

### Example 2 — SQLite schema drift.db (CTX-02)

```sql
-- .planning/agent-drift/drift.db
-- Created by tools/agent-drift/dashboard.py --init

CREATE TABLE IF NOT EXISTS sessions (
  session_id    TEXT PRIMARY KEY,
  started_at    INTEGER NOT NULL,        -- unix seconds
  transcript    TEXT NOT NULL,           -- path to .claude/projects/.../<session>.jsonl
  total_input_tokens  INTEGER,
  total_output_tokens INTEGER,
  total_cache_creation INTEGER,
  total_cache_read     INTEGER
);

CREATE TABLE IF NOT EXISTS commits (
  sha          TEXT PRIMARY KEY,
  author       TEXT NOT NULL,             -- 'human' | 'claude-agent' (parsed from Co-Authored-By)
  committed_at INTEGER NOT NULL,
  subject      TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS violations (
  id           INTEGER PRIMARY KEY AUTOINCREMENT,
  sha          TEXT NOT NULL,             -- foreign key commits.sha
  lint         TEXT NOT NULL,             -- 'accent_lint_fr' | 'no_hardcoded_fr' | 'no_bare_catch'
  file_path    TEXT,
  line_number  INTEGER,
  snippet      TEXT,
  detected_at  INTEGER NOT NULL,
  FOREIGN KEY (sha) REFERENCES commits(sha)
);

CREATE TABLE IF NOT EXISTS context_hits (
  id           INTEGER PRIMARY KEY AUTOINCREMENT,
  session_id   TEXT NOT NULL,
  hit_type     TEXT NOT NULL,             -- 'rule_violation_pre_tool_use' | 'user_prompt_injected'
  rule_id      TEXT,                      -- 'accents' | 'banned_terms' | 'hardcoded_fr' | etc.
  tool_use_index INTEGER,                 -- N-th tool_use in session (0 = before 1st)
  detected_at  INTEGER NOT NULL,
  FOREIGN KEY (session_id) REFERENCES sessions(session_id)
);

CREATE TABLE IF NOT EXISTS golden_runs (
  id           INTEGER PRIMARY KEY AUTOINCREMENT,
  run_at       INTEGER NOT NULL,
  prompt_id    INTEGER NOT NULL,           -- 1..20 golden prompts
  turns_to_correct INTEGER,                -- metric (d) time-to-first-correct-output
  passed_lints TEXT,                       -- JSON array of lint names that passed
  failed_lints TEXT,                       -- JSON array of lint names that failed
  output_excerpt TEXT
);

-- Query examples for the 4 metrics:
-- (a) drift rate = commits agent with ≥1 violation / total commits agent, last 7d
SELECT
  COUNT(DISTINCT c.sha) FILTER (WHERE v.sha IS NOT NULL) * 1.0 /
  COUNT(DISTINCT c.sha) AS drift_rate
FROM commits c
LEFT JOIN violations v ON v.sha = c.sha
WHERE c.author = 'claude-agent'
  AND c.committed_at >= strftime('%s', 'now', '-7 days');

-- (b) context hit rate = sessions with ≥1 pre-1st-tool-use hit / total sessions
SELECT
  COUNT(DISTINCT s.session_id) FILTER (WHERE h.tool_use_index = 0) * 1.0 /
  COUNT(DISTINCT s.session_id) AS context_hit_rate
FROM sessions s
LEFT JOIN context_hits h ON h.session_id = s.session_id
WHERE s.started_at >= strftime('%s', 'now', '-7 days');

-- (c) avg tokens per session
SELECT AVG(total_input_tokens + total_output_tokens + total_cache_creation + total_cache_read)
FROM sessions
WHERE started_at >= strftime('%s', 'now', '-7 days');

-- (d) median turns-to-first-correct-output from golden runs
SELECT ROUND(AVG(turns_to_correct), 2)
FROM golden_runs
WHERE run_at = (SELECT MAX(run_at) FROM golden_runs);
```

### Example 3 — Token cost parsing (CTX-02 metric c)

```python
#!/usr/bin/env python3
# tools/agent-drift/ingest_jsonl.py
# Parses Claude Code session transcripts → inserts into sessions table
# Zero Anthropic API calls — reads local files only.
# Schema verified 2026-04-19 on real transcript:
#   data.message.usage = {
#     "input_tokens": 2,
#     "cache_creation_input_tokens": 31264,
#     "cache_read_input_tokens": 12766,
#     "output_tokens": 31,
#     ...
#   }
import json, sqlite3, sys
from pathlib import Path

TRANSCRIPTS_DIR = Path.home() / '.claude' / 'projects' / '-Users-julienbattaglia-Desktop-MINT'
DB_PATH = Path('.planning/agent-drift/drift.db')

def aggregate_session(jsonl_path: Path):
    totals = {'input': 0, 'output': 0, 'cache_create': 0, 'cache_read': 0}
    started_at = None
    session_id = jsonl_path.stem
    with open(jsonl_path) as f:
        for line in f:
            try:
                obj = json.loads(line)
            except json.JSONDecodeError:
                continue
            if obj.get('type') != 'assistant' and 'message' not in obj:
                if started_at is None and 'timestamp' in obj:
                    # fallback to first timestamped event
                    from datetime import datetime
                    started_at = int(datetime.fromisoformat(obj['timestamp'].replace('Z','+00:00')).timestamp())
                continue
            usage = obj.get('message', {}).get('usage', {})
            totals['input']       += usage.get('input_tokens', 0)
            totals['output']      += usage.get('output_tokens', 0)
            totals['cache_create']+= usage.get('cache_creation_input_tokens', 0)
            totals['cache_read']  += usage.get('cache_read_input_tokens', 0)
            if started_at is None and 'timestamp' in obj:
                from datetime import datetime
                started_at = int(datetime.fromisoformat(obj['timestamp'].replace('Z','+00:00')).timestamp())
    return session_id, started_at, totals

def main():
    conn = sqlite3.connect(DB_PATH)
    for jsonl in TRANSCRIPTS_DIR.glob('*.jsonl'):
        sid, started, t = aggregate_session(jsonl)
        if started is None:
            continue
        conn.execute('''
            INSERT OR REPLACE INTO sessions(
              session_id, started_at, transcript,
              total_input_tokens, total_output_tokens,
              total_cache_creation, total_cache_read
            ) VALUES (?, ?, ?, ?, ?, ?, ?)
        ''', (sid, started, str(jsonl), t['input'], t['output'], t['cache_create'], t['cache_read']))
    conn.commit()
    conn.close()

if __name__ == '__main__':
    main()
```
