**AUTH-03 + AUTH-04 — Call checkAuth() at startup so route guards work on cold start.**

In app.dart, find the AuthProvider creation in MultiProvider (line 974):

```dart
ChangeNotifierProvider(create: (_) => AuthProvider()),
```

Replace with:

```dart
ChangeNotifierProvider(create: (_) {
  final provider = AuthProvider();
  provider.checkAuth();
  return provider;
}),
```

This follows the exact same pattern already used in app.dart for other providers:
- Line 978-981: `ByokProvider()..loadSavedKey()`
- Line 986-989: `CoachProfileProvider()..loadFromWizard()`
- Line 990-994: `LocaleProvider()..load()`
- Line 995-999: `UserActivityProvider()..loadAll()`

**How this fixes AUTH-04 (route guards):** The GoRouter redirect (line 158-191) calls `context.read<AuthProvider>().isLoggedIn`. Before this fix, `isLoggedIn` was always `false` on cold start because `checkAuth()` never ran. After this fix, `checkAuth()` reads the JWT from SecureStorage and sets `_isLoggedIn = true` if valid. The `notifyListeners()` call at the end of `checkAuth()` triggers GoRouter to re-evaluate its redirect, so even if the router evaluates before checkAuth completes, it will re-evaluate after.

- Do NOT modify auth_provider.dart — checkAuth() is already fully implemented and correct.
- Do NOT touch any other providers in the MultiProvider block.

| T-03-01 | Information Disclosure | profile_drawer.dart logout | mitigate | Call AuthProvider.logout() which purges tokens, conversations, BYOK keys, coach memory, analytics — prevents cross-account data bleed |
| T-03-02 | Elevation of Privilege | GoRouter redirect | mitigate | checkAuth() at startup ensures isLoggedIn reflects actual JWT state — prevents unauthenticated access to protected routes |
