1. Import `route_planner.dart`, `coach_profile_provider.dart`, and `provider` package at the top of the file.
2. Change `_buildRouteSuggestion` signature from `static Widget _buildRouteSuggestion(Map<String, dynamic> p)` to `static Widget _buildRouteSuggestion(Map<String, dynamic> p, BuildContext context)`. Update the caller in the `build` switch case to pass `context`.
3. After reading `final prefill = p['prefill'] as Map<String, dynamic>?;` (existing line ~91), add Flutter-side fallback logic:

```dart
// Flutter-side prefill fallback via RoutePlanner
Map<String, dynamic>? mergedPrefill = prefill;
try {
  final profileProvider = context.read<CoachProfileProvider>();
  final profile = profileProvider.profile;
  if (profile != null) {
    final intent = p['intent'] as String?;
    if (intent != null) {
      final planner = RoutePlanner(
        registry: MintScreenRegistry(),
        profile: profile,
      );
      final decision = planner.plan(intent);
      if (decision.prefill != null && decision.prefill!.isNotEmpty) {
        // Merge: backend prefill wins on conflict
        mergedPrefill = {
          ...decision.prefill!,
          if (prefill != null) ...prefill,
        };
      }
    }
  }
} catch (_) {
  // Profile or RoutePlanner not available — use backend prefill only
}
```

4. Pass `mergedPrefill` instead of `prefill` to `RouteSuggestionCard`.
5. Also pass `isPartial` based on whether mergedPrefill is null or empty: `isPartial: mergedPrefill == null || mergedPrefill.isEmpty`.
