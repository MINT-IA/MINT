### New UI Pattern: Prefill Badge on Text Fields

**Problem:** TextControllers are pre-populated but users cannot distinguish auto-filled from manually entered values.

**Solution:** When a field is pre-filled from CoachProfile, display a `SmartDefaultIndicator` badge adjacent to the field label. Use confidence level from `FormPrefillService` data source mapping:
- `estimated` source → confidence 0.25 (grey badge, "~Estimé")
- `userInput` source → confidence 0.60 (grey badge, "~Estimé" — still shown to communicate the value came from profile, not this form)
- `certificate` source → confidence 0.95 (hide badge — treat as confirmed)

**Layout for pre-filled field:**
```
Row(
  children: [
    Text('Capital LPP', style: MintTextStyles.bodySmall()),  // field label
    SmartDefaultIndicator(source: 'Depuis ton avoir LPP', confidence: 0.60),
  ]
)
TextFormField(controller: _lppTotalCtrl, ...)  // pre-filled value, fully editable
```

**Badge placement rule:** Badge sits on the label row (top of field), never overlaid on the input itself. Input field remains fully editable — pre-fill is a default, not a lock.

## Screen-by-Screen Interaction Contract

### All 6 Calculator Screens (common pattern)

**Phase:** Simulator screens (Category B from `docs/DESIGN_SYSTEM.md §2B`)

**initState prefill flow:**
1. Read `GoRouterState.of(context).extra` as `Map<String, dynamic>?`
2. If `extra` contains a `'prefill'` key, extract the sub-map
3. For each known field key, populate the corresponding `TextEditingController`
4. Set `_isPreFilled = true` to trigger badge rendering
5. Pattern: `simulator_3a_screen.dart:71-79` (existing reference implementation)

**Fields pre-filled per screen** (from CONTEXT.md decisions):

| Screen | Route | Prefill keys from CoachProfile |
|--------|-------|-------------------------------|
| Rente vs Capital | `/rente-vs-capital` | `avoirLppTotal`, `tauxConversion`, `salaireBrutMensuel`, `ageRetraite` |
| Pilier 3a | `/pilier-3a` | `salaireBrutMensuel`, `canton` |
| Hypothèque | `/hypotheque` | `salaireBrutMensuel`, `epargneLiquide`, `avoirLppTotal` |

| 3a rétroactif | `/3a-retroactif` | `salaireBrutMensuel`, `canton` |
| EPL | `/epl` | `avoirLppTotal`, `salaireBrutMensuel` |

|---------|----------|-------------|

| Primary CTA (simulation trigger) | `calculatorCtaSimuler` | "Simuler" |
| Write-back snackbar | `profileUpdatedSnackbar` | "Profil mis à jour" |
| Prefill badge — label tooltip title | `prefillBadgeTitle` | "Valeur pré-remplie" |
| Prefill badge — source label | `prefillBadgeSourceLabel` | "Source" |
| Prefill badge — estimated source description | `prefillSourceEstimated` | "Estimation depuis ton profil MINT" |
| Prefill badge — user input source description | `prefillSourceUserInput` | "Valeur que tu as déclarée à MINT" |
| Prefill badge — "Préciser" action | `prefillBadgePrecise` | "Préciser ce chiffre" |
| Partial data warning (existing) | `routeSuggestionPartialWarning` | "Données incomplètes — les résultats seront estimés" (already exists) |
| Empty state — no prefill available | `calculatorNoPrefillBody` | "Saisis les valeurs pour lancer le calcul. MINT complètera automatiquement les prochaines fois." |
| Error state — write-back failed | `profileUpdateErrorSnackbar` | "Mise à jour du profil impossible. Réessaie depuis ton profil." |

**Notes:**
- "Calculer" and "Simuler" already exist on most screens — verify before adding new i18n keys (grep for hardcoded strings first, then add to ARB only for gaps)
- Non-breaking space before `%`, `!`, `?`, `:`, `;` — mandatory per CLAUDE.md §6 voice rules
- No banned terms: "garanti", "certain", "optimal", "meilleur" are forbidden
- Tone: informal "tu", educational, never prescriptive

### Empty State
**When:** Calculator opened directly (not via RouteSuggestionCard) and CoachProfile has no relevant data for the fields.

Heading: none (no empty state heading — input fields are already visible with placeholder hints)
Body: `calculatorNoPrefillBody` — shown as a subtle grey `bodySmall` hint below the first empty input group, not as a blocking modal.

### Error State
**When:** Write-back to CoachProfile fails after simulation.

Display: `profileUpdateErrorSnackbar` shown as a standard snackbar with `MintColors.error` background. Duration: 3000ms. No action button — user recovers by visiting their profile screen.

### Destructive Actions
None in this phase. Calculator inputs are editable defaults; no delete, reset, or irreversible operation is exposed.

## Interaction States

| State | Visual treatment |
|-------|-----------------|
| Field pre-filled from profile | `SmartDefaultIndicator` badge on label row; field fully editable |
| Field pre-filled from certificate (confidence 0.95+) | No badge — treat as confirmed input; field still editable |
| Partial prefill (`isPartial: true` on RouteSuggestionCard) | Warning banner in amber (`MintColors.warningBg` + `MintColors.warning` text) before CTA |
| Calculation in progress | No full-screen loader; inline loading on result area only (existing `MintLoadingSkeleton` pattern from `rente_vs_capital_screen.dart`) |
| Write-back in progress | No visible indicator — fire-and-forget after result renders |
| Write-back success | `profileUpdatedSnackbar` — 2500ms, `MintColors.primary` background, `MintColors.white` text |
| Write-back failure | `profileUpdateErrorSnackbar` — 3000ms, `MintColors.error` background, `MintColors.white` text |
| Plan staleness triggered | No UI in this phase — `FinancialPlanProvider` detects staleness silently; Phase 4's `FinancialPlanCard` stale state handles the visual (already implemented) |
