## Spacing Scale

All values from `MintSpacing` in `lib/theme/mint_spacing.dart`. No new tokens added for this phase.

| Token | Dart constant | Value | Usage in this phase |
|-------|--------------|-------|---------------------|
| xs | MintSpacing.xs | 4px | Icon-to-label gap, chip separator |
| sm | MintSpacing.sm | 8px | Subtitle gap below title, chip list separator |
| md | MintSpacing.md | 16px | PremierEclairageCard internal padding |
| lg | MintSpacing.lg | 24px | Horizontal screen padding (IntentScreen, HomeScreen) |
| xl | MintSpacing.xl | 32px | Below title before chip list, above microcopy |
| xxl | MintSpacing.xxl | 48px | — |
| xxxl | MintSpacing.xxxl | 64px | Top breathing room on IntentScreen (existing) |

Exceptions:
- Chip tile vertical padding: 16px (MintSpacing.md) — touch target minimum 48px total height achieved via text + padding, consistent with existing `_IntentChipTile` implementation.
- PremierEclairageCard: 20px vertical internal padding — between xxxl (64) and xl (32) is not a standard token. Use `MintSpacing.xl` (32px) top/bottom padding inside the card instead to stay on the 8-point scale.

## Typography

All styles from `MintTextStyles` in `lib/theme/mint_text_styles.dart`. Roles declared for this phase only.

**Weight contract:** exactly 2 weights — w400 (regular) for all body text, labels, microcopy, and chips; w700 (bold) for all headings, display numbers, and CTA labels.

**Size contract:** exactly 4 sizes — 14px, 16px, 20px, 32px.

| Role | MintTextStyles method | Size | Weight | Font | Line Height |
|------|-----------------------|------|--------|------|-------------|
| Premier eclairage number | `displayMedium()` | 32px | w700 | Montserrat | 1.15 |
| IntentScreen title | `headlineLarge()` | 32px | w700 | Montserrat | 1.15 |
| Card title (PremierEclairageCard) | `headlineSmall()` | 20px | w700 | Montserrat | 1.2 |
| CTA label ("Comprendre") | `titleMedium()` | 16px | w700 | Inter | 1.3 |
| Intent chip label | `bodyLarge()` | 16px | w400 | Inter | 1.5 |
| Card body / explanation text | `bodyMedium()` | 14px | w400 | Inter | 1.5 |
| Microcopy / disclaimer | `bodySmall()` | 14px | w400 | Inter | 1.4 |

Rationale: `displayMedium` for the premier eclairage number creates the "moment of revelation" without competing with the home screen's own data hierarchy. `displayLarge` (48px) is reserved for full hero/pulse screens per the design system rule ("one dominant number per screen"). IntentScreen title uses the same 32px/w700 token as the display number — both are dominant single-screen headings. Microcopy uses 14px (same size as body) to stay within the 4-size limit, differentiated from body by color (`MintColors.textMuted`) rather than size.

## Component Inventory

New components introduced by this phase:

### PremierEclairageCard

**File:** `apps/mobile/lib/widgets/onboarding/premier_eclairage_card.dart`
**Placement:** MintHomeScreen, Section 0 (inserted above existing Chiffre Vivant GPS section, first-visit only)
**Dismissal:** Tappable "×" in top-right corner. Auto-dismisses when `hasSeenPremierEclairage` flag is set (after user explores ≥1 simulator).

Visual structure:
```
┌─────────────────────────────────────────────┐
│ ░░░  [Number: displayMedium, textPrimary]    │  ← saugeClaire left accent bar (4px wide)
│ ░░░  [Title: headlineSmall]               ×  │
│      [Subtitle: bodyMedium, textSecondary]   │
│                                              │
│      [ Comprendre → ]                        │  ← accent fill button, full-width
└─────────────────────────────────────────────┘
```

- Corner radius: 12px (consistent with chip tiles)
- Left accent bar: 4px wide, full card height, `MintColors.saugeClaire`, borderRadius left only
- Internal padding: `MintSpacing.md` (16px) all sides
- Dismiss icon: `Icons.close`, 18px, `MintColors.textMuted`, positioned top-right
- Entrance animation: FadeTransition + SlideTransition from y=+0.08 offset, duration `MintMotion.slow` (600ms), curve `MintMotion.curveEnter`
- Dismissal animation: FadeTransition out, duration `MintMotion.standard` (300ms), curve `MintMotion.curveExit`
- Skeleton/loading: Show `_PremierEclairageCardSkeleton` (shimmer-free: use `MintColors.appleSurface` shimmer with opacity animation) while ChiffreChocSelector computes. Duration ≤100ms in practice — show skeleton only if computation exceeds 50ms.

### intent_router.dart (new service, no UI)

Mapping service, not a widget. See 03-CONTEXT.md D-01 and D-02 for full mapping table.

### Modified: IntentScreen post-tap behavior

No visual change to the screen itself. Change is in `_onChipTap`: after existing persistence calls, add `IntentRouter.resolve(chipKey)` and write `goalIntentTag` to `CapMemoryStore`. Navigate to `/home?tab=0` instead of `/home?tab=1`.

### Modified: CoachChatScreen opener

When `selectedOnboardingIntent` is non-null, replace generic `_computeKeyNumber()` silent opener with intent-aware opener message. No visual change to the chat UI itself.

## Screen Transitions

Per ROADMAP.md Phase 8 (UXP-04), full `CustomTransitionPage` fade is a Phase 8 concern. However, the PremierEclairageCard itself introduces a first-visit reveal:

- IntentScreen → `/home?tab=0`: standard GoRouter page transition (no change in this phase)

Copywriting rules (from CLAUDE.md §6 + VOICE_SYSTEM.md):
- Informal "tu" throughout
- Non-breaking space (`\u00a0`) before `!`, `?`, `:`, `;`, `%` — mandatory
- No banned terms: never "garanti", "optimal", "meilleur", "parfait"
- No "chiffre choc" — use "premier éclairage" (CLAUDE.md legacy note)
- Conditional framing on numbers: if showing an estimate, add "environ" or "en moyenne"
- Max label length: 40 characters for chip labels, 60 for card titles

Empty state (no PremierEclairageCard shown — returning sessions):
- Not applicable: card is only shown on first session. Subsequent sessions show normal MintHomeScreen without Section 0.

Error state (ChiffreChocSelector returns null):
- Heading: "Calcul non disponible" (`premierEclairageCardErrorTitle`)
- Body: "Complète ton profil pour voir ton premier aperçu." (`premierEclairageCardErrorBody`)
- CTA: "Personnaliser" → navigates to `/onboarding/quick-start`
- Do NOT show the card at all if both number and fallback are unavailable — skip Section 0 silently.

Destructive actions in this phase: none.
