# Phase 4 — Plan Generation: UI Design Contract

> Visual and interaction contract for Phase 4 frontend work.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

> Aujourd'hui tab) and `PlanPreviewCard` (inline widget in coach chat). Both are
> read-only, calculator-backed, and adaptive to profile changes.

## Component Specifications

### FinancialPlanCard (MintHomeScreen insertion)

Position: between ChiffreVivantCard (Section 1) and the signal/lever sections. Shown only when `FinancialPlanProvider.hasPlan == true`. Hidden entirely if no plan — no empty state card.

```
Container:
  background: MintColors.white
  borderRadius: 16px (MintSpacing.md — radiusLg)
  border: MintColors.border at 0.5 alpha
  shadow: none (border is sufficient — rule: never both)
  padding: MintSpacing.md (16px) all sides

Layout (top to bottom):
  Row: [goal label (titleMedium, textSecondary)] [stale badge if stale]
  SizedBox: MintSpacing.xs (4px)
  Text: monthly target CHF (displayMedium, textPrimary) — e.g. "850 CHF / mois"
  SizedBox: MintSpacing.sm (8px)
  Text: target date (bodyMedium, textMuted) — e.g. "Objectif: avril 2028"
  SizedBox: MintSpacing.md (16px)
  LinearProgressIndicator: height 6px, value 0.0 (no check-ins yet)
  SizedBox: MintSpacing.sm (8px)
  Row: [caption "0% atteint"] [Spacer] [TextButton "Voir le détail"]
```

Stale state: amber badge ("Profil modifié — recalculer") replaces goal label suffix. "Recalculer" replaces "Voir le détail" as the CTA.

### PlanPreviewCard (inline in coach chat)

Appears in the chat message bubble after `GENERATE_FINANCIAL_PLAN` tool call resolves.

```
Container:
  background: MintColors.appleSurface (#F5F5F7)
  borderRadius: 12px (radiusMd)
  padding: MintSpacing.md (16px)

Layout (top to bottom):
  Text: goal description (titleMedium, textPrimary) — e.g. "Acheter un appartement en 2028"
  SizedBox: MintSpacing.sm (8px)
  Text: monthly target (displayMedium, textPrimary) — e.g. "850 CHF / mois"
  SizedBox: MintSpacing.md (16px)
  Divider: MintColors.border at 0.5 alpha
  SizedBox: MintSpacing.sm (8px)
  Text: "Jalons" label (bodySmall, textMuted, w600)
  SizedBox: MintSpacing.xs (4px)
  ListView: 4 milestone rows (date + target CHF + description) — bodyMedium, textSecondary
  SizedBox: MintSpacing.md (16px)
  Text: coach narrative (bodyLarge, textSecondary) — 2-3 sentences max
  SizedBox: MintSpacing.md (16px)
  Text: disclaimer (micro, textMuted, italic) — "Outil éducatif — ne constitue pas un conseil. Sources: LIFD art. 38, LPP art. 14."
```

Monte Carlo confidence: show low/mid/high as 3 small text values below the monthly amount — bodySmall, using `MintColors.trajectoryPrudent` / `MintColors.trajectoryBase` / `MintColors.trajectoryOptimiste`.

### Plan Detail View (expandable section — Claude's Discretion choice)

Implement as an expandable section within MintHomeScreen (not a new route), using `AnimatedSize` + `AnimatedOpacity`. Avoids navigation overhead and keeps the Aujourd'hui tab self-contained.

```
When expanded, insert below FinancialPlanCard:
  Container: same card style as FinancialPlanCard
  Layout:
    Text: "Jalons trimestriels" (titleMedium, textSecondary)
    4 milestone rows: [Q date (bodyMedium, textMuted)] [CHF target (bodyMedium, textPrimary, w600)] [description (bodySmall, textMuted)]
    SizedBox: MintSpacing.md
    Text: confidence bands row (bodySmall): "Bas: X CHF | Moyen: Y CHF | Haut: Z CHF"
    SizedBox: MintSpacing.sm
    Text: disclaimer (micro, textMuted, italic)
```

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Coach sends `GENERATE_FINANCIAL_PLAN` | `ChatToolDispatcher` normalizes payload → `PlanPreviewCard` renders inline in chat → `FinancialPlanProvider` persists plan → `FinancialPlanCard` appears on MintHomeScreen on next render |
| Tap "Voir le détail" | Expands inline detail section via `AnimatedSize` (no navigation push) — button label toggles to "Masquer le détail" |
| Tap "Recalculer" (stale state) | Opens coach tab with pre-seeded goal text "Recalcule mon plan : [goal]" via `CoachEntryPayload` — user confirms, plan regenerates |
| Profile salary changes | `FinancialPlanProvider` detects hash mismatch → marks plan stale → amber badge appears without re-computation — no auto-regeneration |
| Progress bar | Always 0% in Phase 4 (check-ins are Phase 5). Render bar at 0% visually — do not hide it. It signals what's coming. |
| No plan state | `FinancialPlanCard` section is invisible — `Visibility(visible: false)` not `if` removal, to preserve layout slot |

## Copywriting Contract

All strings go into ARB files (`lib/l10n/`) — no hardcoded French text in Dart files.

| Element | French copy | i18n key |
|---------|-------------|----------|
| Primary CTA | "Voir le détail" | `planCard_ctaDetail` |
| Stale CTA | "Recalculer" | `planCard_ctaRecalculate` |
| Stale badge | "Profil modifié — recalculer" | `planCard_staleBadge` |
| Goal label prefix | "Mon plan :" | `planCard_goalPrefix` |
| Target date label | "Objectif : {date}" | `planCard_targetDate` |
| Milestones heading | "Jalons trimestriels" | `planCard_milestonesHeading` |
| Progress caption | "{percent}% atteint" | `planCard_progressCaption` |
| Confidence bands | "Bas : {low} CHF  |  Moyen : {mid} CHF  |  Haut : {high} CHF" | `planCard_confidenceBands` |
| Detail toggle (open) | "Voir le détail" | `planCard_ctaDetail` |
| Detail toggle (close) | "Masquer le détail" | `planCard_ctaHide` |
| Empty state | Section hidden — no copy needed | — |
| Disclaimer | "Outil éducatif — ne constitue pas un conseil financier (LSFin). Sources\u00a0: LIFD art.\u00a038, LPP art.\u00a014." | `planCard_disclaimer` |
| Error state | "Impossible de générer le plan. Réessaie en décrivant ton objectif au coach." | `planCard_errorBody` |
| Recalculate prompt (sent to coach) | "Recalcule mon plan : {goalDescription}" | `planCard_recalculatePrompt` |

Banned terms (never appear in any copy): "garanti", "certain", "optimal", "meilleur", "parfait", "conseillez". Use "pourrait", "envisager", "selon le scénario moyen".

Non-breaking spaces mandatory before: `!`, `?`, `:`, `;`, `%` — use `\u00a0` in ARB values.

## Numbers & Data Display Rules

| Data | Format | Example |
|------|--------|---------|
| Monthly CHF amount | `CHFFormatter` — `X'XXX CHF / mois` | `850 CHF / mois` |
| Milestone targets | `CHFFormatter` — `X'XXX CHF` | `21'250 CHF` |
| Target date | `DateFormat('MMMM yyyy', 'fr')` | `avril 2028` |
| Milestone date | `DateFormat('QQQ yyyy', 'fr')` | `T3 2027` |
| Confidence bands | `CHFFormatter` no `/mois` suffix | `Bas : 710 CHF` |
| Progress percent | `${(value * 100).round()}%` | `0%` |

## Compliance Requirements (mandatory on all plan surfaces)

Per CLAUDE.md §6 and DESIGN_SYSTEM.md §2B:

1. `disclaimer` field rendered in `micro` style at card footer — "Outil éducatif, ne constitue pas un conseil, LSFin"
2. `sources` rendered in same `micro` block — LIFD art. 38 and LPP art. 14 minimum
3. Monte Carlo confidence bands (low/mid/high) shown when `confidenceLevel < 0.70`
4. No guaranteed return language — always use "scénario moyen" framing
5. "Recalculer" flow must not auto-send — user must explicitly tap Send in coach
