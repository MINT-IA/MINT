# Requirements: MINT v1.0 UX Journey

**Defined:** 2026-04-05
**Core Value:** A user opens MINT and within 3 minutes receives a personalized, surprising insight about their financial situation — then knows exactly what to do next.

## v1 Requirements

Requirements for UX Journey milestone. Each maps to roadmap phases.

### Cleanup

- [ ] **CLN-01**: Duplicate service pairs resolved (canonical imports only, no re-exports masquerading as separate services)
- [ ] **CLN-02**: Orphan routes triaged — each of 67 canonical routes is either live, redirected, or explicitly archived
- [ ] **CLN-03**: Dead screens removed (screens with no route pointing to them)

### Tool Dispatch

- [ ] **TDP-01**: Coach tool calls (show_fact_card, show_score_gauge, route_to_screen, etc.) reach the UI and render appropriate widgets
- [ ] **TDP-02**: ChatToolDispatcher created — parses tool call markers from backend response, dispatches to appropriate UI handler
- [ ] **TDP-03**: RoutePlanner suggestions rendered as tappable cards in chat (not just logged "not yet handled")
- [ ] **TDP-04**: CoachRichWidgetBuilder uses LLM tool decisions (not keyword matching fallback) to choose display widgets

### Onboarding Pipeline

- [ ] **ONB-01**: User completes intent selection and sees first personalized insight (premier eclairage) within 3 minutes
- [ ] **ONB-02**: IntentScreen selection triggers CapSequenceEngine to generate relevant first journey
- [ ] **ONB-03**: JourneyTrigger connects onboarding intent to appropriate calculator/insight flow
- [ ] **ONB-04**: Post-onboarding landing is contextual (based on selected intent), not generic home screen

### Plan Generation

- [ ] **PLN-01**: Coach generates a personalized financial plan from user's declared goal (e.g., "acheter un appartement dans 3 ans" -> monthly savings target, timeline, milestones)
- [ ] **PLN-02**: Generated plan includes chiffered milestones (monthly amount, intermediate targets, projected outcome)
- [ ] **PLN-03**: Plan is visible as a persistent artifact (not buried in chat history) — accessible from profile or Aujourd'hui tab
- [ ] **PLN-04**: Plan adapts when user profile changes (salary increase, new goal, life event)

### Suivi & Check-in

- [ ] **SUI-01**: Monthly check-in is proactively triggered (nudge/notification when it's time)
- [ ] **SUI-02**: Check-in flow is conversational (coach asks "combien as-tu versé ce mois?") not a form
- [ ] **SUI-03**: Progress visualization shows plan vs. reality (PlanRealityCard wired and visible on Aujourd'hui)
- [ ] **SUI-04**: Coach references past check-ins ("le mois dernier tu avais versé X, ce mois...") using cross-session memory
- [ ] **SUI-05**: Streak and engagement metrics visible to user (not just tracked silently)

### Calculator Wiring

- [ ] **CAL-01**: Calculator screens pre-fill all known fields from CoachProfile (never ask what MINT already knows)
- [ ] **CAL-02**: RoutePlanner.prefill decisions passed through GoRouter extras to calculator constructors
- [ ] **CAL-03**: Calculator results feed back into CoachProfile (bidirectional data flow)

### Life Event Journeys

- [ ] **LEJ-01**: firstJob journey verified end-to-end (onboarding intent -> relevant calculators -> insight -> plan -> suivi)
- [ ] **LEJ-02**: housingPurchase journey verified end-to-end (intent -> EPL/mortgage calc -> savings plan -> monthly check-in)
- [ ] **LEJ-03**: newJob journey verified end-to-end (intent -> salary comparison -> LPP transfer check -> 3a optimization)
- [ ] **LEJ-04**: Each journey has an integration test (not just unit tests on individual components)

### UX Polish

- [ ] **UXP-01**: Signal cards on Aujourd'hui tab animate on data change (AnimatedSwitcher, not hard refresh)
- [ ] **UXP-02**: ConfidenceScore visible to user with explanation of what improves it
- [ ] **UXP-03**: Explorer hubs show ReadinessGate state (greyed/locked items vs. available based on profile completeness)
- [ ] **UXP-04**: Navigation transitions feel guided (CustomTransitionPage on journey screens, not default push)

## v2 Requirements

Deferred to next milestone. Tracked but not in current roadmap.

### Coach Depth

- **CDH-01**: Multi-step reasoning chains in coach (tool_use -> execute -> re-call LLM loop)
- **CDH-02**: Coach personality refinement per life event context (not just regional tone)
- **CDH-03**: Proactive weekly recap pushed to user (not just available on demand)

### Data Enrichment

- **DAT-01**: OCR capture of LPP certificate for automatic profile enrichment
- **DAT-02**: Open Banking integration for real-time transaction tracking
- **DAT-03**: Backend RAG vector store live for semantic memory retrieval

### Advanced Journeys

- **ADV-01**: All 18 life events with verified end-to-end journeys
- **ADV-02**: Couple mode journeys (Julien + Lauren simultaneous planning)
- **ADV-03**: Cross-border / frontalier specific journeys

## Out of Scope

| Feature | Reason |
|---------|--------|
| New financial calculators | Existing 8 calculators cover all v1 scenarios |
| Backend API restructuring | Focus is frontend assembly; backend is correct |
| Voice AI | Phase 3 roadmap (post-journey foundation) |
| B2B / institutional features | Phase 4 roadmap |
| Money movement / transactions | Compliance: read-only forever |
| Real-time bank sync | v2 requirement (Open Banking) |
| Product recommendations | Compliance: no-advice rule (LSFin) |
| Social comparison | Compliance: banned (compare to own past only) |
| Multi-agent coach | v2 (1 process with 6 modes is sufficient for v1) |
