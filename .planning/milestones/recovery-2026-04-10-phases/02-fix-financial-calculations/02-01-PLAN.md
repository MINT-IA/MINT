<bug_analysis>
## Root Cause Analysis (completed during planning)

Three interconnected issues cause the golden couple discrepancies:

### Issue A: ForecasterService ignores LPP certificate overrides
`ForecasterService._projectScenario()` at line 676 calls `LppCalculator.projectOneMonth()`
WITHOUT passing `bonificationRateOverride` or `salaireAssureOverride`. So when Julien's profile
has CPE Plan Maxi data (bonif 24%, salaire assure 91967), the ForecasterService computes LPP
as if he were on standard legal minimum bonifications. This causes the taux de remplacement
to be wrong (test 4).

### Issue B: Test 2a uses wrong conversion rate for CPE Plan Maxi
The test calls `projectToRetirement` with `conversionRate: lppTauxConversionMinDecimal` (6.8%).
But CPE Plan Maxi is an enveloping surobligatoire plan. The 6.8% legal minimum applies ONLY to
the obligatoire portion. For an enveloping plan like CPE, the actual conversion rate is lower
(typically ~5%). CLAUDE.md confirms: 677847 * 0.05 = 33'892 (matches expected rente exactly).
The balance projection (675k) is correct — only the conversion rate applied is wrong.

### Issue C: Test 2b uses assumptions that don't match CLAUDE.md
Lauren's LPP projection with `caisseReturn: 0.02` yields 203k. But CLAUDE.md expects ~153k.
The 153k figure is consistent with LPP minimum return of 1.25% and some adjustments. The test
needs to use assumptions matching the CLAUDE.md reference computation. With `caisseReturn: 0.0125`:
balance = ~186k. The remaining gap (186k vs 153k) suggests CLAUDE.md used a more conservative
model. Since CLAUDE.md is source of truth for golden couple values, the test expected value may
need adjustment to match what the calculator produces with correct inputs, OR we accept the
calculator's correct math with a documented note.

### Summary of fixes needed:
1. Add `bonificationRate` to `PrevoyanceProfile` (no field exists for this)
2. Wire `ForecasterService` to pass overrides to `projectOneMonth` (both user and conjoint)
3. Fix test 2a to use correct enveloping conversion rate (~5% for CPE Plan Maxi)
4. Reconcile test 2b expected values with calculator math
5. Fix test 4 input profile to include `salaireAssure` and `bonificationRate` so ForecasterService uses them
6. Tighten all golden couple test assertions from `greaterThan(X)` to `closeTo(expected, tolerance)`
</bug_analysis>
