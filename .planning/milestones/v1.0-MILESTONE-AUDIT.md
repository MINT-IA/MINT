## Blockers (2)

### BLOCKER 1: Plan Generation Pipeline Broken (PLN-01, PLN-02)

`generate_financial_plan` is NOT registered in `services/backend/app/services/coach/coach_tools.py`. The LLM cannot produce this tool call. Additionally, `PlanGenerationService.generate()` is never imported by any production file — the calculator-backed plan computation is dead code.

**Impact:** The entire plan generation flow (coach generates plan → PlanPreviewCard in chat → FinancialPlanCard on Aujourd'hui) never triggers. Data layer is complete and correct; the wire is broken at both ends.

**Fix:** Register `generate_financial_plan` in `COACH_TOOLS` and wire `PlanGenerationService.generate()` into the `WidgetRenderer` dispatch path.

### BLOCKER 2: Sequential Check-in Degraded (SUI-02)

`planned_contributions` is not included in `BackendCoachService.chat()` request body AND not in `_PROFILE_SAFE_FIELDS` whitelist in `coach_chat.py`. Claude receives the `_CHECK_IN_PROTOCOL` instructions but has no contribution list to iterate over — falls back to generic "how much did you save" instead of per-contribution sequential questions.

**Impact:** Check-in works but is not personalized. SUI-04 (memory reference) IS wired correctly.

**Fix:** Add `plannedContributions` to `BackendCoachService.chat()` body and `planned_contributions` to `_PROFILE_SAFE_FIELDS`.
