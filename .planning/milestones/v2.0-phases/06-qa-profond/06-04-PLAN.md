Create `apps/mobile/test/accessibility/wcag_audit_test.dart` with 3 groups:

**Group 1 — Contrast ratio verification:**
- Import MintColors and extract all color pairs used in v2.0 screens.
- Implement a `contrastRatio(Color foreground, Color background)` helper using WCAG luminance formula: `(L1 + 0.05) / (L2 + 0.05)` where L = relative luminance.
- Test the following critical pairs from MintColors:
  - Primary text on white background >= 4.5:1
  - Secondary text on white background >= 4.5:1
  - Primary text on card background >= 4.5:1
  - White text on primary color >= 4.5:1 (for buttons)
  - Error text on white background >= 4.5:1
  - Hint/subtitle text on white background >= 3.0:1 (large text exception)
- Each pair that fails below threshold = test failure with the actual ratio logged.

**Group 2 — Tap target sizes:**
- Pump key interactive widgets from v2.0: PremierEclairageCard action button, AnticipationSignalCard dismiss/snooze buttons, privacy control screen fact cards, intent chips on intent_screen.
- Use `tester.getSize()` to verify each tappable element is >= 44x44 logical pixels.
- If a widget is smaller, the test fails with the actual size.

**Group 3 — Semantics labels:**
- Pump MintHomeScreen (with mocked providers), find all Semantics widgets.
- Assert that at least the card tap targets have semanticLabel or tooltip.
- Pump intent_screen, assert intent chips have semanticLabel.

Create `apps/mobile/test/accessibility/font_scaling_test.dart`:

**Group 1 — 200% font scaling:**
- For each new v2.0 screen (MintHomeScreen, intent_screen, privacy_control_screen, premier_eclairage_card):
  - Pump with `MediaQuery(data: MediaQueryData(textScaler: TextScaler.linear(2.0)), child: ...)` 
  - Assert no overflow errors via `tester.takeException()` returning null.
  - Use `FlutterError.onError` capture to detect RenderFlex overflow.
- Total: ~4 screens tested at 200% scaling.
