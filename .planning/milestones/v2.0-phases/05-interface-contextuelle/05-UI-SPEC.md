# Phase 5 — UI Design Contract

> Visual and interaction contract for the Interface Contextuelle phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Flutter Material 3 (existing) |
| Preset | not applicable |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, tight grouping within card rows |
| sm | 8px | Between closely related elements (title row icon gap, button row spacing) |
| md | 16px | Card internal padding (all card types), standard vertical gap between card elements |
| lg | 24px | Screen horizontal padding (SafeArea inset), section label to first card |
| xl | 32px | Between card slots in the Aujourd'hui feed |
| 2xl | 48px | Hero stat top breathing room (above chiffre vivant) |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch targets on dismiss/snooze/CTA buttons are 48px minimum height (tapTargetSize constraint), overriding any text-based sizing.

## Typography

This phase uses EXACTLY 4 sizes and 2 weights from the existing MintTextStyles system.

| Role | Size | Weight | Line Height | MintTextStyles Method | Usage in Phase 5 |
|------|------|--------|-------------|----------------------|-------------------|
| Display | 48px | w700 | 1.1 | `displayLarge()` | Hero stat number (chiffre vivant: "4'416 CHF/mois") |
| Heading | 26px | w700 | 1.15 | `headlineLarge()` | Coach opener sentence, "Aujourd'hui" section label |
| Body | 16px | w400 | 1.5 | `bodyLarge()` | Card narrative text, educational fact, action description |
| Label | 14px | w400 | 1.5 | `bodyMedium()` | Card metadata (source ref, date), "Voir plus" expand label, dismiss/snooze text |

Weight mapping:
- **w700** (bold): hero stat number, section headings, card titles, primary CTA text
- **w400** (regular): body content, metadata, secondary actions

## Component Inventory

### Card 1: Hero Stat Card (`HeroStatCard`)

- **Surface:** `MintSurface(tone: blanc)`, padding `MintSpacing.md` (16px)
- **Layout:** Column > [label 14px w400 muted] > [number 48px w700 primary] > [narrative 16px w400 secondary] > [delta badge row]
- **Delta badge:** Inline Row with icon (trending_up/down, 16px) + delta text (14px, success/warning color)
- **Tap behavior:** Entire card tappable, deep-links to relevant simulator via GoRouter
- **Touch target:** Full card surface (well above 48px)

### Card 2: Anticipation Signal Card (existing `AnticipationSignalCard`)

- **Preserved from Phase 4** -- no visual changes
- **Surface:** `MintSurface(tone: porcelaine)`, padding `MintSpacing.md`
- **Layout:** Row[icon 20px + title 14px w700] > fact 14px w400 > source 10px italic > CTA + dismiss/snooze
- **Dismiss:** "Compris" (w700 primary), "Plus tard" (w400 secondary)
- **Touch target:** TextButton with 48px minimum height

### Card 3: Progress/Milestone Card (`ProgressMilestoneCard`)

- **Surface:** `MintSurface(tone: peche)`, padding `MintSpacing.md`
- **Layout:** Column > [icon 24px + title 16px w700] > [progress description 16px w400] > [AnimatedProgressBar]
- **Progress bar:** Existing `AnimatedProgressBar` widget, height 8px, radius 4px, fill color `MintColors.success`
- **Touch target:** Entire card tappable, deep-links to profile enrichment

### Card 4: Action Opportunity Card (`ActionOpportunityCard`)

- **Surface:** `MintSurface(tone: blanc)`, padding `MintSpacing.md`
- **Layout:** Row[icon 20px accent + Column[title 14px w700 + body 16px w400]] > chevron_right icon 20px muted
- **Tap behavior:** Entire card tappable, deep-links to action (scan document, complete profile, review projection)
- **Touch target:** Full card surface, minimum 48px height
- **Chevron:** `Icons.chevron_right_rounded`, 20px, `MintColors.textMuted`

### Card 5: Expandable Overflow (`ContextualOverflow`)

- **Surface:** `MintSurface(tone: craie)`, padding `MintSpacing.md`
- **Collapsed state:** Row["Voir plus" 14px w400 primary + count badge 14px w400 muted + expand_more icon 20px]
- **Expanded state:** AnimatedCrossFade to Column of additional cards (same card types as above)
- **Animation:** `MintMotion.standard` duration (300ms), Curves.easeInOut
- **Touch target:** Expand/collapse row minimum 48px height

### Shared Card Patterns

- **Border radius:** 16px (MintSurface default)
- **Elevation:** 0 (flat cards on warm background, no shadows)
- **Card gap:** `MintSpacing.xl` (32px) between card slots
- **Entry animation:** `MintEntrance` with staggered delay (0ms, 100ms, 200ms, 300ms per card index)
- **Deep-link pattern:** `GestureDetector` + `context.push(route)` via GoRouter (same as Phase 4 anticipation cards)

### Coach Opener

- **Position:** Above card feed, below profile button
- **Style:** `MintTextStyles.headlineLarge()` (26px w700), `MintColors.textPrimary`

## Interaction Contract

### Card Ranking (CTX-02)

- Computed once at app launch via `WidgetsBinding.instance.addPostFrameCallback`
- `priority_score = timeliness * 0.5 + user_relevance * 0.3 + confidence * 0.2`
- Top 4 cards rendered directly; overflow cards behind "Voir plus"
- Max 5 visible slots (4 direct + 1 overflow container)
- Completed action sets card priority to 0 (demoted to bottom or removed)

### Card Dismissal (CTX-05)

- Anticipation cards: "Compris" / "Plus tard" (existing Phase 4 pattern)
- Action cards: completing the action auto-demotes (no explicit dismiss)
- Progress cards: no dismiss (always visible while milestone in progress)
- Hero stat: never dismissible (always slot 1)

### Session Determinism (CTX-02)

- Card order frozen after initial computation
- Dismiss/snooze updates `SharedPreferences` but does NOT re-rank remaining cards mid-session
- Next app launch recalculates fresh ranking

### Deep Links (CTX-04)

| Card Type | Example Route | Pattern |
|-----------|--------------|---------|
| Hero Stat | `/retirement/projection` | Profile-dependent simulator |
| Anticipation | `/simulators/3a` | Signal-specific simulator link |
| Progress | `/onboarding/quick?section=profile` | Profile enrichment |
| Action | `/documents/capture` | Contextual next action |

### Navigation Shell (CTX-06)

- 3-tab shell preserved: Aujourd'hui (index 0) | Coach (index 1) | Explorer (index 2)
- ProfileDrawer (endDrawer) unchanged
- No new tabs, no tab reordering, no drawer modifications

- Never imperative commands ("fais ceci", "tu devrais") in card text or coach opener

## Accessibility Contract

| Criterion | Requirement |
|-----------|-------------|
| Contrast | All text meets WCAG 2.1 AA (4.5:1 minimum). Already verified in MintColors. |
| Touch targets | All interactive elements (cards, buttons, expand toggle) minimum 48px height |
| Screen reader | Each card has `Semantics(label:)` with card type + title. Expand/collapse announces state. |
| Font scaling | Cards use `MediaQuery.textScaler` — no fixed-height containers that clip at 200% scale |
| Motion | `MintEntrance` respects `MediaQuery.disableAnimations` — skips animation when reduced motion enabled |
