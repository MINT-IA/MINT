<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Apple -> backend | Apple identity token transmitted from client, must be verified server-side |
| client -> backend /auth/apple/verify | Identity token could be forged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-11 | Spoofing | /auth/apple/verify | mitigate | Verify Apple JWT issuer (`iss == https://appleid.apple.com`), audience matches app bundle ID, token not expired. Production: validate against Apple's public keys. |
| T-01-12 | Tampering | Apple identity token | mitigate | Token is signed by Apple. Backend decodes and verifies signature (or at minimum issuer+audience for MVP). |
| T-01-13 | Information Disclosure | Apple email | accept | Apple provides email via Sign-In. User can choose to hide email (Apple relay). We store whatever Apple provides. |
</threat_model>
