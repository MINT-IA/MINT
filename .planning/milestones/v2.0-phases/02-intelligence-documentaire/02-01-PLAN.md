<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> extract-vision API | Untrusted image data crosses here (potentially malicious PDF/image) |
| extract-vision -> Claude API | Image sent to external LLM for processing |
| audit log -> database | Metadata stored (must NOT contain image data or PII) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | DocumentAuditLog | mitigate | user_id stored as SHA-256 hash, zero image fields, zero field values in audit model |
| T-02-02 | Information Disclosure | extract-vision endpoint | mitigate | finally-block clears image_base64 from memory; audit_log.deleted_at records deletion time |
| T-02-03 | Tampering | image upload | mitigate | existing 5MB max_length on image_base64, 10/min rate limit, file type validation |
| T-02-04 | Denial of Service | classify_document | accept | classification adds one extra API call; 30s timeout inherited from Vision call; rate limit already at 10/min |
| T-02-05 | Spoofing | classification bypass | mitigate | classify_document fails-open (returns is_financial=True on error) to prevent blocking legitimate users; extraction itself has range checks |
</threat_model>
