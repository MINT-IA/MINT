## Interaction Contracts

### Screen 1: Document Capture (document_scan_screen)

- **Entry**: From coach suggestion chip ("Scanne ton certificat LPP") or Explorer hub deep-link
- **Layout**: Form category (D) -- headline top + document type selector + capture options + sticky bottom area
- **Document type selector**: Horizontal scrollable chips (4 types: certificat LPP, certificat de salaire, attestation 3a, police d'assurance). Selected chip: primary fill, white text. Unselected: surface fill, textSecondary. Each chip 48px height (touch target).
- **Capture options**: 3 tappable cards stacked vertically, each with icon + label + description:
  - Camera (Icons.camera_alt): "Prendre une photo" / "Le plus rapide"
  - Gallery (Icons.photo_library): "Choisir une image" / "Screenshot ou photo existante"
  - PDF (Icons.picture_as_pdf): "Importer un PDF" / "Document numerique"
  - Each card: MintSurface, 48px min height, md (16px) internal padding, radiusMd (12px)
- **File validation (DOC-10 client-side)**: Accept image/* and application/pdf only. Max 10MB. On invalid: inline error (bodyMedium, error color) below capture area -- "Format non supporte. Utilise une photo ou un PDF."
- **Processing state**: Full-screen overlay with MintLoadingState + "Analyse en cours..." message. No cancel button (extraction is fast, < 10s).
- **Backend rejection (DOC-10 server-side)**: If document classified as non-financial, show inline error card: warning icon + "Ce document ne semble pas etre un document financier suisse." + "Essaie avec un certificat LPP, de salaire, 3a, ou une police d'assurance." (bodyMedium, warning background)
- **Error state**: MintErrorState with "Reessayer" if backend call fails
- **Exit**: Success navigates to extraction_review_screen with ExtractionResult

### Screen 3: Extraction Review (extraction_review_screen)

- **Entry**: From document capture after successful extraction
- **Layout**: Form category (D) -- SliverAppBar + header + confidence badge + warnings + field cards + sticky confirm CTA
- **Header**: headlineLarge title + bodyMedium subtitle with field count and review count (already exists)
- **Overall confidence badge**: Pill badge with icon + percentage text. Color per confidence mapping. Already exists -- no changes.
- **LPP plan type warning (DOC-04)**: If LPP document and plan type is "1e", show prominent warning banner above field cards:
  - Background: `MintColors.warningBg` (#FFF3E0)
  - Border: `MintColors.warning` at 0.3 opacity, 12px radius
  - Icon: Icons.warning_amber_outlined, 18px, warning color
  - Text (bodyMedium): "Plan 1e detecte. Pas de taux de conversion garanti -- projection en capital uniquement."
  - Position: below overall confidence badge, above field cards
- **Cross-field coherence warning (DOC-05)**: If obligatoire + surobligatoire does not match total (> 5% tolerance), show warning banner:
  - Same visual style as LPP warning (warning background, warning border, warning icon)
  - Text (bodyMedium): "Les montants obligatoire et surobligatoire ne correspondent pas au total. Verifie les valeurs."
  - Position: below LPP warning (if both present), above field cards
- **Field cards** (per field): MintSurface with:
  - Top row: confidence icon (18px, badge color) + field label (bodyMedium, textSecondary) + confidence percentage pill (bodyMedium, badge color)
  - Value row: bodyLarge, textPrimary, editable on tap (opens inline TextField)
  - Source text row (DOC-09): bodyMedium italic, textMuted, shows source_text passage. Prefixed with "Source : "
  - Red fields (confidence < threshold): highlighted border (error color at 0.3 opacity), inline prompt "Verifie cette valeur" (bodyMedium, error color)
- **Confirm CTA**: Sticky bottom, primary button "Confirmer et enrichir mon profil" (full-width, 48px height)
- **Loading state**: MintLoadingState replaces content while ProfileEnrichmentDiff processes
- **Error state**: MintErrorState with "Reessayer"
- **Exit**: Success navigates to document_impact_screen

### Screen 5: Document Impact (document_impact_screen)

- **Entry**: After profile enrichment confirmation
- **Layout**: Hero category (A) -- porcelaine background, animated confidence circle, premier eclairage
- **Confidence circle**: Animated ring (Apple Watch style, already exists) showing old -> new confidence. Delta displayed as displayLarge (+N points). Circle fill color: success.
- **Premier eclairage (DOC-07)**: Below confidence circle. Generated from newly extracted data using 4-layer engine:
  - Layer 2 (human translation): bodyLarge, textSecondary, 1-2 sentences
  - Layer 3 (personal perspective): bodyLarge, textPrimary, personalized insight
  - Layer 4 (questions to ask): bodyMedium list with bullet prefix
- **Disclaimer**: bodyMedium italic, textMuted, bottom -- "Outil educatif. Ne constitue pas un conseil financier (LSFin)."
- **CTA**: "Continuer" (primary button) navigates back to coach or home
- **Document deletion confirmation**: No visual indication to user (happens silently in finally block per COMP-04). Audit log is backend-only.
- **Loading state**: MintLoadingState while premier eclairage generates
- **Error state**: If premier eclairage fails, show extracted data summary instead (graceful degradation) with "Ton profil a ete enrichi" heading + field summary cards

## Accessibility Contracts (DOC-01, QA-08)

| Rule | Value | Applies To |
|------|-------|------------|
| Minimum tap target | 48px x 48px | Capture option cards, document type chips, CTA buttons, field edit tap areas |
| Color contrast (text) | >= 4.5:1 on background | All text (verified: textMuted #737378 = 4.6:1 on white) |
| Color contrast (large text) | >= 3:1 | displayLarge, headlineLarge |
| Confidence not color-only | Icon shape changes per level (check_circle / warning_amber / error_outline) | All confidence badges |
| Font scaling | Supports up to 200% | All text via MediaQuery.textScaleFactor |
| Screen reader | Semantic labels on confidence badges ("Confiance elevee 95 pourcent"), capture options, field cards | All interactive and informational elements |
| Focus order | Top-to-bottom, left-to-right | All screens |
| Error announcement | Errors and warnings announced to screen reader via Semantics | Rejection messages, coherence warnings, field verification prompts |
| Camera permission | System dialog, fallback to gallery/PDF if denied | document_scan_screen |
