## Critical Gap: Document → Biography Bridge

**P0 — Breaks core value proposition.**

When a user confirms document extraction in `extraction_review_screen.dart`, the fields update `CoachProfile` but are NOT written as `BiographyFact` entries. The biography database is permanently empty in production.

**Impact cascade:**
- Privacy screen ("Ce que MINT sait de toi") always empty
- Coach opener always generic (never "Ton salaire a augmenté...")
- Salary increase anticipation trigger never fires
- Action opportunity detector always suggests "scan a document"

**Fix:** Add `context.read<BiographyProvider>().addFact(...)` calls in `_onConfirmAll()` for each confirmed extraction field.

## Compliance Gap

**P1 — `/documents/premier-eclairage` returns LLM text without ComplianceGuard validation.**

The coach chat endpoint passes output through ComplianceGuard. The document insight endpoint does not. Fix: add `ComplianceGuard.validate()` call before returning response.

## Platform Gap

**P2 — Patrol tests are manual-gate only.**

QA-04/QA-05 Patrol integration tests exist as scripts but CI lacks emulator infrastructure. Documented as manual gate in `.github/workflows/patrol.md`.
