# Plan de lancement — MINT

> Ce document évolue. Il capture la stratégie de go-to-market.

## Phases

### Phase 0 — Closed Alpha (actuel)
- Toi + proches testeurs
- TestFlight iOS
- Discord pour les retours
- Objectif : valider les parcours core (retraite, budget, 3a)

### Phase 1 — Closed Beta
- 50-100 utilisateurs invités
- Critères : résidents suisses, 25-65 ans, motivés par la prévoyance
- Canal : invitation par email / Discord
- Objectif : NPS > 40, retention J7 > 50%
- **Prérequis compliance** : DPO désigné, DPA signés, DPIA validée

### Phase 2 — Open Beta / Soft Launch
- App Store + Google Play (accès libre)
- Landing page avec waitlist
- Newsletter mensuelle
- Objectif : 1000 utilisateurs, validation product-market fit
- **Prérequis** : Sentry/Crashlytics, monitoring backend, support email

### Phase 3 — Launch public
- PR / presse spécialisée (PME Magazine, Bilan, Le Temps)
- Partenariats caisses de pension
- Référencement conseiller.ch / comparis.ch
- Objectif : 10'000 utilisateurs

## Discord — Rôle dans le lancement

Discord sert de :
1. **Canal de feedback direct** — les testeurs remontent les bugs et suggestions
2. **Communauté early adopters** — engagement, rétention, ambassadeurs
3. **Support** — réponses rapides aux questions
4. **Transparence** — partager les updates de développement (webhook GitHub → #dev-updates)

Ce que Discord ne fait PAS :
- Gestion de projet (pas de kanban, pas de deadlines)
- Documentation (pas de structure, pas de recherche)
- CRM (pas de suivi contacts/leads)
